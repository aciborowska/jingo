My apologies in advance for potentially my most pedantic JIRA ever... 

The interactive console and docs do not describe how to specify units for the cache size of db-cache-size. Recomend updating the docs to specify what the acceptable units are. These appear to be b (byte), (kilobyte), kib (kibibyte), gb (gigabyte) and gib (gibibyte). 

It would be much more clear if the tools/docs used the industry standard notations of B, KiB and GiB instead of b, kib and gib.  

Technically speaking, b is bit and not byte, gb is gigabit and not gigabyte etc. The notations kib and gib (all lower case) don't officially exist.

For reference, the following command sets a 2048MiB db cache:


./dsconfig set-backend-prop --port 4444 --bindDN "CN=Directory Manager" --bindPassword cangetindj --backend-name userRoot --set db-cache-size:2147483648b --trustAll --no-prompt

The following values will also set the same thing, 2048MiB:


db-cache-size:2gib
db-cache-size:2097152kib
