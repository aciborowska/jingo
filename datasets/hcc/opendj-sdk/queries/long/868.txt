Adding attributes to the Referential Integrity plugin fails with "Server is unwilling to perform (53)" error, when using both dsconfig and ldapmodify. The only way to successfully add attributes is to stop OpenDJ, modify config.ldif and add "ds-cfg-attribute-type: memberof" to "dn: cn=Referential Integrity,cn=Plugins,cn=config",and start OpenDJ.

Steps to reproduce:
1. Configure opendj and enable referential integrity
2. Add an attribute to the referential integrity plugin with dsconfig or ldapmodify.

I was specifically attempting to add the memberof attribute after extending the schema for OpenAM and had already applied opendj_config_schema.ldif and opendj_user_schema.ldif. I then attempted to run ldapmodify with opendj_plugin.ldif from openam which will enable RI and add the attribute.
