If you want to use multiple profiles such as :

--profile am-config:6.5.2 \

--profile am-cts:6.5.2 \

Then you must specify unique values for --set am-cts/baseDn 

If you use the same, then one of the backends will fail to start with an error such as 

[30/Sep/2019:11:26:03 -0500] category=CONFIG severity=ERROR msgID=116 msg=An error occurred while trying to initialize a backend loaded from class org.opends.server.backends.jeb.JEBackend with the information in configuration entry ds-cfg-backend-id=cfgStore,cn=Backends,cn=config: An error occurred while attempting to register the base DNs [dc=example,dc=edu] in the Directory Server: Unwilling to Perform: Unable to register base DN dc=example,dc=edu with the Directory Server for backend cfgStore because that base DN is already registered for backend amCts. This backend will be disabled

Additionally, you cannot use baseDN this will fail with the following warning.

Error: Parameter ‘baseDn’ is provided without any profile specification which is not allowed when multiple profiles are used.
