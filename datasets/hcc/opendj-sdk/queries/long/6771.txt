When the DS is configured to use /dev/stdout for any logger, the DS Extract hangs attempting to gather the file.

Example:

 

./supportextract  --bindDN "cn=Directory Manager" --bindPassword `cat passfile` --outputDirectory /opt/instances/data

...
2019-08-19 16:14:35 INFO  Collecting backend statistics
2019-08-19 16:14:35 INFO  - cfgStore: total jdb files 1
2019-08-19 16:14:35 INFO  - Adding je.info.0, je.config.csv and je.stat.csv
2019-08-19 16:14:35 INFO  Collecting the log files
2019-08-19 16:14:35 INFO  * /dev/stdout *

The Extract needs to throw an exception when this is encountered.

 

Workaround:

Use --maxLogFiles 0 to skip gathering all logs.  The logs must be gathered and sent separately.

Note: the use of /dev/stdout for any logger is not supported in all versions including and up to 6.5.x.
