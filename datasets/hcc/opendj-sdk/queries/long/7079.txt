We support taking JE backend offline backups when the server is running but we do not support this for any other type of backend because this may lead to corrupted backup files.
 Nevertheless, dsbackup allows to do that regardless of the backend type, see this example:

$ ./opendj/dsconfig list-backends --offline 
Backend      : Type : enabled : base-dn             : confidentiality-enabled
-------------:------:---------:---------------------:------------------------
adminRoot    : ldif : true    : cn=admin data       : -
dsEvaluation : je   : true    : "dc=example,dc=com" : false
monitorUser  : ldif : true    : uid=Monitor         : -
rootUser     : ldif : true    : uid=admin           : -
$ ./opendj/bin/start-ds
$ ./opendj/bin/dsbackup create --offline --backupDirectory bak --backendName monitorUser
[03/04/2020:19:03:36 +0200] category=TOOLS seq=0 severity=INFO msg=Starting backup for backend 'monitorUser'
[03/04/2020:19:03:36 +0200] category=TOOLS seq=1 severity=INFO msg=Backup completed for backend 'monitorUser' with backup ID 'monitorUser_20200403170336608'



Note that this was also an issue with the old backup tool.

This issue can be closed once dsbackup create prevents users from taking non-JE backend backups when the server is running.
