If you add a cipher that is bogus or otherwise not supported by the JVM, dsconfig will allow the change to take place.

 


The Crypto Manager was modified successfully

The equivalent non-interactive command-line is:
dsconfig set-crypto-manager-prop \
          --set ssl-cipher-suite:TLS_INDECIPHERABLE_CIPHER \
          --hostname localhost \
          --port 4444 \
          --bindDn cn=Directory\ Manager \
          --bindPasswordFile pass \
          --trustAll \
          --no-prompt

Press RETURN to continue

        Property                     Value(s)
        -----------------------------------------------------------------------
    1)  key-wrapping-transformation  RSA/ECB/OAEPWITHSHA-1ANDMGF1PADDING
    2)  ssl-cert-nickname            ads-certificate
    3)  ssl-cipher-suite             TLS_INDECIPHERABLE_CIPHER
    4)  ssl-encryption               true
    5)  ssl-protocol                 Uses the default set of SSL protocols
                                     provided by the server's JVM.

    ?)  help
    f)  finish - apply any changes to the Crypto Manager
    c)  cancel
