Following the changes to the tools, the config/MakeLDIF/example.template is missing objectClass attributes. As a result, when you generate LDIF, the objectClass attributes are missing from the top-level entries:


dn: dc=example,dc=com
dc: example

dn: ou=People,dc=example,dc=com
ou: People

dn: uid=user.0,ou=People,dc=example,dc=com
objectClass: top
objectClass: person
objectClass: organizationalPerson
objectClass: inetOrgPerson
givenName: Aaccf
sn: Amar
cn: Aaccf Amar
...



As a result, when you import the generated LDIF, the command fails with an error message such as the following:


The import has been aborted because the entry 'uid=user.0,ou=People,dc=example,dc=com' does not have a parent entry
