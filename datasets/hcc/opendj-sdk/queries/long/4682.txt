When generating the LDAP Schema Reference after recent syntax additions, I noticed that the X-SUBST notation is not supported:

[INFO] Unknown extra property: X-SUBST=[1.3.6.1.4.1.1466.115.121.1.27]
[INFO] Unknown extra property: X-SUBST=[1.3.6.1.4.1.1466.115.121.1.27]
[INFO] Unknown extra property: X-SUBST=[1.3.6.1.4.1.36733.2.1.3.1]
[INFO] Unknown extra property: X-SUBST=[1.3.6.1.4.1.36733.2.1.3.1]


When encountering X-SUBST, the generation code should create links to the substitute syntax, rather than OIDs.
