Given an LDAP entry having a JSON based attribute "session":



dn: uid=bjensen,...
objectClass: top
...
uid: bjensen
mail: bjensen@example.com
...
session: {"access_token":"789","expires_in":59,"token_type":"Bearer","refresh_token":"012"}



Which is exposed using Rest2Ldap as:



{
  "uid": "bjensen",
  "mail": "mail=bjensen@example.com",
  "session": {
    "access_token": "789",
    "expires_in": 59,
    "token_type": "Bearer", 
    "refresh_token": "012"
  }
}



Then it should be possible to perform a query over Rest2Ldap having a filter which queries internal fields of the LDAP "session" attribute:



GET /path/to/users?_queryFilter=uid eq "bjensen" and session/access_token eq "789"
