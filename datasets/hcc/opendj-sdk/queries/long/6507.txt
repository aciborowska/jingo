Backup offline saves every backend which supports backup whereas backup online and saves enabled backends only.

Cedric Tran-Xuan and Ludovic Poitou have agreed that the offline and online commands should have the same behavior and both should ignore disabled backends.

This issue can be closed when backup offline ignores disabled backends: 

	with --backUpAll option: skip the disabled backend
	with --backendId option: fail if at least one of the provided backend is disabled.



Reproduce cases
Backup a disabled backend with --backUpAll option

	Disable appData

$ dsconfig set-backend-prop \
          --backend-name appData \
          --set enabled:false \
          --hostname localhost \
          --port 4444 \
          --bindDn uid=admin \
          --trustAll \
          --bindPassword password \
          --no-prompt






	Backup all online: skips appData, this is the correct behavior

$ backup --backupDirectory /path/to/backup/dir -h localhost -p 4444 -X --bindDn "uid=admin" -w password --backUpAll
$ ls /path/to/backup/dir
adminRoot      ads-truststore monitorUser    rootUser       schema         tasks






	Backup all offline: does not skip appData, this is the bug

$ backup --backupDirectory /path/to/backup/dir --offline --backUpAll
$ ls /path/to/backup/dir
adminRoot      ads-truststore appData        monitorUser    rootUser       schema         tasks





Backup a disabled backend with --backendId option

	Disable appData

$ dsconfig set-backend-prop \
          --backend-name appData \
          --set enabled:false \
          --hostname localhost \
          --port 4444 \
          --bindDn uid=admin \
          --trustAll \
          --bindPassword password \
          --no-prompt






	Backup appData online: stops with an error, this is the correct behavior

$ backup --backupDirectory /path/to/backup/dir -h localhost -p 4444 -X --bindDn "uid=admin" -w password --backendId appData
[02/Aug/2019:11:48:54 +0200] severity="ERROR" msgCount=1 msgID=org.opends.messages.tool-261 message="None of the Directory Server backends are configured with the requested backend ID "appData""
Backup task 20190802114854560 did not complete successfully






	Backup appData offline: succeeds, this is the bug

$ backup --backupDirectory /path/to/backup/dir --offline --backendId appData
[02/08/2019:11:51:34 +0200] category=TOOLS seq=2 severity=INFO msg=The backup process completed successfully
