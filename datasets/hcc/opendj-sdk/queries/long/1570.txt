First, setup a replication topology.

Add an entry:

$ ldapmodify -p 1500 -D "cn=Directory Manager" -w admin -a <<END_OF_COMMAND_INPUT
dn: cn=A1,dc=example,dc=com
objectclass:top
objectclass:organizationalperson
objectclass:inetorgperson
objectclass:person
sn:User
cn:Test User
description:1
description:2
mail:bla@example.com
telephonenumber:+33165990803
END_OF_COMMAND_INPUT



Here is the output of running an persistent search with changesOnly=true before the add:

$ ldapsearch -p 1500 -D "cn=Directory Manager" -w admin -C ps:all -T -b "dc=example,dc=com" "(objectclass=*)"
# Persistent search change type: add
dn: cn=A1,dc=example,dc=com
objectClass: person
objectClass: organizationalperson
objectClass: inetorgperson
objectClass: top
description: 1
description: 2
cn: Test User
cn: A1
telephonenumber: +33165990803
sn: User
mail: bla@example.com

# Persistent search change type: modify
dn: dc=example,dc=com
objectClass: domain
objectClass: top
dc: example



The 'modify' change to the entry with dn 'dc=example,dc=com' look dubious.

Running a persistent search with the operational attributes before the add shows what this change was about:

$ ldapsearch -p 1500 -D "cn=Directory Manager" -w admin -C ps:all -T -b "dc=example,dc=com" "(objectclass=*)" '+' '*'
# Persistent search change type: add
dn: cn=A1,dc=example,dc=com
objectClass: person
objectClass: organizationalperson
objectClass: inetorgperson
objectClass: top
description: 1
description: 2
cn: Test User
cn: A1
telephonenumber: +33165990803
sn: User
mail: bla@example.com
ds-sync-hist: dn:00000148834fbb447ad600000002:add
entryUUID: 636140c6-4c67-42bf-b48a-62a55f0eaa82
createTimestamp: 20140917111153Z
creatorsName: cn=Directory Manager,cn=Root DNs,cn=config

# Persistent search change type: modify
dn: dc=example,dc=com
objectClass: domain

ds-sync-state: 00000148834fbb447ad600000002
entryUUID: 54c89c45-327b-3d38-8a2e-b7814de651a4

The 'modify' change to the entry with dn 'dc=example,dc=com' was about 'ds-sync-generation-id' and 'ds-sync-state' attributes.
Note: 'ds-sync-hist' attribute also appear for the entry with dn 'cn=A1,dc=example,dc=com'.
