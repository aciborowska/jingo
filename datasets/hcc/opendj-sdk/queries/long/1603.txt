Problem: 
User is able to stop or restart the server via stop-ds/stop-ds -R while he hasn't got sufficient privileges to initiate the shutdown.

How to reporduce:

	Install OpenDJ nightly (2.7.0) with baseDN: dc=com
	Add new Root DN. using following command:

./bat/ldapmodify -h ondrej -p 389 -D cn=directory\ manager -w pwd --defaultAdd 
dn: cn=Aroot Manager,cn=Root DNs,cn=config
objectClass: person
objectClass: organizationalPerson
objectClass: inetOrgPerson
objectClass: ds-cfg-root-dn-user
objectClass: top
ds-cfg-alternate-bind-dn: cn=Aroot
givenName: Aroot
cn: Aroot Manager
sn: Manager
userpassword: PrivsRule
ds-pwp-password-policy-dn: cn=Root Password Policy,cn=Password Policies,cn=config
ds-rlim-lookthrough-limit: 0
ds-rlim-time-limit: 0
ds-rlim-size-limit: 0


	Add privilege (in fact disallow to shutdown the server) for default Directory Manager:

./bat/ldapmodify -h ondrej -p 389 -D "cn=Aroot" -w PrivsRule --defaultAdd
dn: cn=Directory Manager,cn=Root DNs,cn=config
changetype: modify
add: ds-privilege-name
ds-privilege-name: -server-shutdown


	Stop the server via default root user using following command:

./bat/stop-ds -h ondrej -p 5444 -D "cn=directory manager" -w pwd -X




	ON LINUX
	
		printed message "You do not have sufficient privileges to initiate a Directory Server shutdown" => OK
	
	
	ON WINDOWS
	
		the server is stopping => BUG
	
	



5. Stop the server via new root user using following command:

./bat/stop-ds -h ondrej -p 5444 -D "cn=Aroot" -w PrivsRule -X



	on both platforms the instance is stopping => OK
