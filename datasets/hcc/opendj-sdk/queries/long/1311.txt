Setting 'db-txn-no-sync' property without prior changing 'db-txn-write-no-sync' leads to


dsconfig set-backend-prop -h localhost -p 4444 -D 'cn=Directory Manager' -j PATH_TO_PWD_FILE --trustAll --backend-name userRoot --set db-txn-no-sync:true --no-prompt

The Local DB Backend could not be modified because of the following reason:

    *  [LDAP: error code 53 - Entry
       ds-cfg-backend-id=userRoot,cn=Backends,cn=config cannot be modified
       because one of the configuration change listeners registered for that
       entry rejected the change:  Configuration attributes
       ds-cfg-db-txn-no-sync and ds-cfg-db-txn-write-no-sync are mutually
       exclusive and cannot be both set at the same time]



Shouldn't dsconfig show properties mentioned in the ref-guide instead of the attributes from the config.ldif?
