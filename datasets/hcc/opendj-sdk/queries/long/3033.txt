https://backstage.forgerock.com/#!/docs/opendj/3/admin-guide#assign-pwp-for-branch

This example states that it will set the password policy for the 'ou=People,dc=example,dc=com' tree.

It will actually set the password policy for the whole dc=example,dc=com tree.

I think the example should either be:



dn: cn=People Password Policy,dc=example,dc=com
objectClass: top
objectClass: subentry
objectClass: pwdPolicy
cn: People Password Policy
pwdAttribute: userPassword
pwdLockout: TRUE
pwdMaxFailure: 3
pwdFailureCountInterval: 300
pwdLockoutDuration: 300
pwdAllowUserChange: TRUE
pwdSafeModify: TRUE
subtreeSpecification: { base "ou=people" }



OR - create a global password policy and use:



dn: cn=Assign People Password Policy,dc=example,dc=com
objectClass: top
objectClass: subentry
objectClass: extensibleObject
objectClass: collectiveAttributeSubentry
cn: Assign People Password Policy
ds-pwp-password-policy-dn;collective: cn=People Password Policy,cn=Password Policies,cn=config
subtreeSpecification: { base "ou=people" }



But not both at once.
