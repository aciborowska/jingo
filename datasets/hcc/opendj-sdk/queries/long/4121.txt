proxy-server setup tool subcommand can always be used with either the trustStorePassword or the trustStorePasswordFile options regardless a truststore file (either usePkcs12TrustStore, useJceksTrustStore or useJavaTruststore) option is provided on the command line.

For example, the command below just works fine, the provided trust store password is ignored by the tool:

$ ./setup proxy-server -D "cn=Directory Manager" -w "password" -p 1389 -h localhost --adminConnectorPort 4444 -Z 1636  --acceptLicense --proxyUserBindDN cn=proxy --proxyUserBindPassword proxyPassword  --staticPrimaryServer localhost:1389 --trustStorePassword ignored

Validating parameters..... Done
Configuring certificates..... Done
Configuring server..... Done

To see basic server status and configuration, you can launch
/path/to/opendj/bin/status



This is minor since it does not prevent a proxy server to be setup but it is not optimal regarding user experience
