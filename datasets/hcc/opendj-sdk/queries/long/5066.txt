The default Error Log Account Status Notification Handler does not log password reset.

To reproduce the issue:

# Install a directory server with Example.ldif sample data:
/path/to/opendj/setup directory-server \
    --rootUserDN "cn=Directory Manager" \
    --rootUserPassword password \
    --monitorUserPassword password \
    --hostname opendj.example.com \
    --ldapPort 1389 \
    --ldapsPort 1636 \
    --httpPort 8080 \
    --httpsPort 8443 \
    --adminConnectorPort 4444 \
    --baseDN dc=example,dc=com \
    --ldifFile Example.ldif \
    --acceptLicense

# Configure the error logger to record info messages:
/path/to/opendj/bin/dsconfig set-log-publisher-prop \
    --hostname opendj.example.com \
    --port 4444 \
    --bindDN "cn=Directory Manager" \
    --bindPassword password \
    --publisher-name "File-Based Error Logger" \
    --set default-severity:info \
    --trustAll \
    --no-prompt

# Reset a password:
/path/to/opendj/bin/ldappasswordmodify \
    --hostname opendj.example.com \
    --port 1636 \
    --bindDn cn=Directory\ Manager \
    --bindPassword password \
    --authzID dn:uid=bjensen,ou=people,dc=example,dc=com \
    --useSSL \
    --trustAll

# Check the errors log:
tail /path/to/opendj/logs/errors
...
[07/May/2018:18:10:36 +0200] category=EXTENSIONS severity=INFORMATION msgID=637 msg=Handshake for 'Administration Connector': cipher requires the alias(es) '[server-cert]' to contain key(s) of type(s) '[EC]'.
[07/May/2018:18:13:17 +0200] category=EXTENSIONS severity=INFORMATION msgID=637 msg=Handshake for 'LDAPS': cipher requires the alias(es) '[server-cert]' to contain key(s) of type(s) '[EC]'.

So you do see info level log messages, but no message about the change in account status for Babs Jensen.
