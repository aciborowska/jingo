1) Set up replication with shortish purge delay.
2) Perform some updates.
3) Wait for purge to occur.
4) Read firstChangeNumber.
5) Search for changes great than a number less than the firstChangeNumber.

Expected result: all changes from firstChangeNumber through to lastChangeNumber.
Actual result: only the last change, i.e. whose changeNumber is lastChangeNumber.


./bin/ldapsearch -p 1389 -D "cn=Directory Manager" -w password -b "" -s base '(objectClass=*)' lastChangeNumber firstChangeNumber
dn: 
lastChangeNumber: 28851
firstChangeNumber: 27419

./bin/ldapsearch -p 1389 -D "cn=Directory Manager" -w password -b "cn=changelog" '(changeNumber>=27419)' dn|head
