When setting a value in config.ldif file using an EL expression (https://backstage.forgerock.com/docs/ds/5/admin-guide#expressions), it cannot be later overwritten using dsconfig command.

Steps to replicate:

	Setup DS, stop the server
	Change config.ldif file to use an EL expression ex.:


dn: cn=HTTP OAuth2 OpenAM,cn=HTTP Authorization Mechanisms,cn=config
changetype: modify
replace: ds-cfg-openam-token-info-url
ds-cfg-openam-token-info-url: ${"expression.url:8080/openam/tokeninfo"}
-


	Start the server for config to take effect
	Try to change the value using dsconfig


dsconfig set-http-authorization-mechanism-prop \
          --mechanism-name "HTTP OAuth2 OpenAM" \
          --set token-info-url:http://openam.example.com:8080/openam/oauth2/tokeninfo \
          --hostname localhost.localdomain \
          --port 4444 \
          --bindDn cn=Directory\ Manager \
          --bindPassword ****** \
          --trustAll \
          --no-prompt


Expected behavior
It should overwrite the expression in config.ldif with the value set by user through dsconfig
Current behavior
Throws error:


The HTTP OAuth2 OpenAM Authorization Mechanism could not be modified due to a
communications problem: No Such Attribute: An unexpected error occurred while
attempting to modify configuration entry cn=HTTP OAuth2 OpenAM,cn=HTTP
Authorization Mechanisms,cn=config as a child of entry cn=HTTP OAuth2
OpenAM,cn=HTTP Authorization Mechanisms,cn=config: No Such Attribute: Unable
to remove one or more values from attribute 'ds-cfg-openam-token-info-url'
because at least one of the attributes does not exist in the entry
