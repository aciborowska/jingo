In the replication domain property, if you add a fractional-exclude list that contains a mix of literals/OIDs it seems that it is not possible to remove them afterwards.

testcase: fractional-replication: main.fractional-replication.config: mixing literal/OIDs

Example:


./dsconfig set-replication-domain-prop --provider-name "Multimaster Synchronization" --domain-name "dc=europe,dc=com" --reset fractional-include --add fractional-exclude:*:mail,jpegPhoto,0.9.2342.19200300.100.1.60 --add fractional-exclude:inetOrgPerson:jpegPhoto,0.9.2342.19200300.100.1.60 --add fractional-exclude:2.16.840.1.113730.3.2.2:jpegPhoto,0.9.2342.19200300.100.1.60 -h localhost -p 1444 -D "cn=directory manager" -w "secret12" -n -X

./dsconfig get-replication-domain-prop --provider-name "Multimaster Synchronization" -s --domain-name "dc=europe,dc=com" --property fractional-include --property fractional-exclude -h localhost -p 1444 -D "cn=directory manager" -w "secret12" -n -X
fractional-exclude	*:mail,jpegPhoto,0.9.2342.19200300.100.1.60	2.16.840.1.113730.3.2.2:jpegPhoto,0.9.2342.19200300.100.1.60	inetOrgPerson:jpegPhoto,0.9.2342.19200300.100.1.60
fractional-include

./dsconfig set-replication-domain-prop --provider-name "Multimaster Synchronization"    --domain-name "dc=europe,dc=com" --reset fractional-include --remove fractional-exclude:*:mail,jpegPhoto --remove fractional-exclude:*:0.9.2342.19200300.100.1.60 --remove fractional-exclude:InetOrgPerson:0.9.2342.19200300.100.1.60 -h localhost -p 1444 -D "cn=directory manager" -w "secret12" -n -X
The value *:mail,jpegPhoto for the fractional-exclude property does not exist
