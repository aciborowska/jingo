If hostSource is entered incorrectly in replication initialization command, error returned indicates that remote server could not be connected to, but hostServer is the local server.

Steps:
1. enable replication between two OpenDJ instances
2. attempt to initialize replication, but type hostSource fqdn incorrectly

./bin/dsreplication initialize --baseDN "dc=openam,dc=forgerock,dc=org" --adminUID admin --adminPassword password --hostSource tst-amsfo1.internal.forerock.com --portSource 4444 --hostDestination tst-amsfo2.internal.forgerock.com --portDestination 4444 -X -n



Expected Results:
Could not connect to the local server tst-amsfo1.internal.forerock.com:4444.
or even
Could not connect to the host server tst-amsfo1.internal.forerock.com:4444.

Actual Results:
Could not connect to the remote server tst-amsfo1.internal.forerock.com:4444.
