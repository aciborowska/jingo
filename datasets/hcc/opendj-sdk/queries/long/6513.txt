Restore offline restores every backend which support backup whereas its online counterpart only restores enabled backends.

Cedric Tran-Xuan and Ludovic Poitou have agreed that the offline and online commands should have the same behavior and both should ignore disabled backends.

This issue can be closed when restore offline terminates with an error message when the backend is disabled.

Reproduce case

	Backup appData

$ backup --backupDirectory /path/to/backup/dir -h localhost -p 4444 -X --bindDn "uid=admin" -w password --backendId appData






	Disable appData

$ dsconfig set-backend-prop \
--backend-name appData \
--set enabled:false \
--hostname localhost \
--port 4444 \
--bindDn uid=admin \
--trustAll \
--bindPassword password \
--no-prompt






	Restore appData online: stops with an error, this is the correct behavior

$ restore --backupDirectory /Users/cyril/deploy/testBu2 -h localhost -p 4444 -X --bindDn "uid=admin" -w password
[02/Aug/2019:11:40:36 +0200] severity="ERROR" msgCount=1 msgID=org.opends.messages.tool-20022 message="Cannot find an enabled backend with ID appData"
Restore task 20190802114036345 did not complete successfully






	Restore appData offline: succeeds, this is the bug

$ restore --backupDirectory /Users/cyril/deploy/testBu2 --offline
[02/08/2019:11:42:37 +0200] category=UTIL seq=0 severity=INFO msg=Restored backup file: 00000000.jdb (size 286397)
