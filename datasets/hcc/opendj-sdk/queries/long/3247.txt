We are using the Maven release plugin to automate the release process.
This plugin replaces the version number in all the pom.xml files.
But for the MSI and SRV4 packages, the plugin is not able to change the version number because of the profile activation:


        <profile>
            <id>opendj-svr4</id>
            <activation>
                <os><family>SunOS</family></os>
            </activation>
            <modules>
                <module>opendj-svr4</module>
            </modules>
        </profile>

        <profile>
            <id>opendj-msi</id>
            <activation>
                <os><family>Windows</family></os>
            </activation>
            <modules>
                <module>opendj-msi</module>
            </modules>
        </profile>



As we are releasing from a Linux platform, so these two profiles are not activated...

After a release, we have to update manually these two projects.
