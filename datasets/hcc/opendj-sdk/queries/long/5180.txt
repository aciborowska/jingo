(Optional) Remove the backend database and the unused LDIF files for the default superuser account, cn=Directory Manager.
dsconfig delete-backend --hostname opendj.example.com --port 4444 --bindDN "cn=Directory Manager" --bindPassword password --backend-name altRootUser --trustAll --no-prompt
rm -rf /path/to/opendj/db/rootUser
In this example, cn=Alternate DM now has the same rights as the original superuser.
The example shows the removing of the AlternateDM back end as opposed to the Directory Manager backend which is being described as being replaced by the Alternate DM

 

Likely the intended instruction should  read:
dsconfig delete-backend --hostname opendj.example.com --port 4444 --bindDN "cn=Alternate DM" --bindPassword password --backend-name rootUser --trustAll --no-prompt
rm -rf /path/to/opendj/db/rootUser
