When starting DS with JDK 11.0.1 through 11.0.3, the DS will hang if -Xlog GC parameters when a file=<windows drive letter + file path> parameter is used.

Example:

	Setup a DS 7.0 instance with --doNotStart
	Copy in a new java.properties file
	Attempt to start-ds



java.properties:

default.java-home=C:\Program Files\Java\jdk-11.0.3

start-ds.java-args=-server -Xlog:gc:file=C:\70\opendj1\logs\windows11-gc.log

Startup:  (Select-Strings used to demonstrate the paths in the java properties file)

PS C:\70> cp "Z:\java.properties" C:\70\opendj1\config\

PS C:\70> Select-String "^default.java-home=" C:\70\opendj1\config\java.propertiesopendj1\config\java.properties:61:default.java-home=C:\Program Files\Java\jdk-11.0.3

PS C:\70> Select-String "start-ds.java-args=" 
opendj1\config\java.properties:65:start-ds.java-args=-server -Xlog:gc:file=C:\70\opendj1\logs\windows11-gc.log

PS C:\70> C:\70\opendj1\bat\start-ds
<hangs here>

Notes:

	This is not a DS issue.
	This is a known issue with all JDK 11 versions up to and including 11.0.3.
	The issue is fixed in  11.0.4



See the following OpenJDK bug

	JDK-8215398 -Xlog option usage => Invalid decorator '\temp\app_cds.log'.

The only workaround for this is to omit the driveletter: in the file= path.

Example java.properties: (the C: is removed)

start-ds.java-args=-server -Xlog:gc:file=\70\opendj1\logs\windows11-gc.log
