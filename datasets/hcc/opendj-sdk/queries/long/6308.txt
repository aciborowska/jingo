The docs have not caught up yet to the change for OPENDJ-5866.

At present, the script setting up replication in https://ea.forgerock.com/docs/ds/getting-started/#setup-repl hangs indefinitely during the doc tests. Here's the content of the script (there's already a first-ds instance running from https://ea.forgerock.com/docs/ds/getting-started/#install ):

# Unpack files for a second, replica server:
cd ~/Downloads && unzip ~/Downloads/opendj-7.0.0-SNAPSHOT.zip && mv opendj /path/to/replica

# Set up a second, replica server configuring only the necessary ports:
/path/to/replica/setup \
 --serverID second-ds \
 --rootUserDn uid=admin \
 --rootUserPassword password \
 --hostname localhost \
 --ldapPort 11389 \
 --adminConnectorPort 14444 \
 --baseDN dc=example,dc=com \
 --acceptLicense

# Prepare to replicate data:
dsreplication \
 configure \
 --adminUID admin \
 --adminPassword password \
 --baseDN dc=example,dc=com \
 --host1 localhost \
 --port1 4444 \
 --bindDN1 uid=admin \
 --bindPassword1 password \
 --replicationPort1 8989 \
 --host2 localhost \
 --port2 14444 \
 --bindDN2 uid=admin \
 --bindPassword2 password \
 --replicationPort2 18989 \
 --trustAll \
 --no-prompt

# Begin replicating data:
dsreplication \
 initialize \
 --adminUID admin \
 --adminPassword password \
 --baseDN dc=example,dc=com \
 --hostSource localhost \
 --portSource 4444 \
 --hostDestination localhost \
 --portDestination 14444 \
 --trustAll \
 --no-prompt 

Notice that the docs have not yet been updated to use the deployment key on the second server when setting it up.
