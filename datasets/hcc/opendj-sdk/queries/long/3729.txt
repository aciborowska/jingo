The makeldif has functionality to override constant in template file using -c/--constant parametr.
The syntax is  constant_name=constant_value, but the problem arise when the constant_value contains an equal sign "=". 

I wasn't able to force the value a string using double or single quotes and I also tried to use backslash to escape it.

Example of command:

./bin/makeldif -o output.ldif -c suffix=dc=example,dc=cz config/MakeLDIF/example.template 
./bin/makeldif -o output.ldif -c suffix="dc=example,dc=cz" config/MakeLDIF/example.template 
./bin/makeldif -o output.ldif -c suffix="dc\=example,dc\=cz" config/MakeLDIF/example.template 



Current behavior:

./bin/makeldif -o output.ldif -c suffix=dc=example,dc=cz config/MakeLDIF/example.template 
Unable to parse a constant argument expecting name=value but got
suffix=dc=example,dc=cz

./bin/makeldif -o output.ldif -c suffix="dc=example,dc=cz" config/MakeLDIF/example.template 
Unable to parse a constant argument expecting name=value but got
suffix=dc=example,dc=cz

./bin/makeldif -o output.ldif -c suffix="dc\=example,dc\=cz" config/MakeLDIF/example.template 
Unable to parse a constant argument expecting name=value but got
suffix=dc\=example,dc\=cz




Expected behavior:
A successful use of new constant value during generating the ldif file.
