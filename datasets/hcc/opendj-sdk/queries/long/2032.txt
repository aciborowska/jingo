Until implementation of issue OPENDJ-1791, dbtest tool should return more accurate message when running against persistit backend.

Scenario:
************
1. configure a server with persistit backend

$ ./opendj/setup --cli -D cn=myself -w password -p 11389 --adminConnectorPort 11444 -b "dc=com" -d 10 -t persistit -n


2. run for instance dbtest list-index-status (same problem with other subcommands)

$ ./opendj/bin/dbtest list-index-status -n userRoot -b dc=com
None of the Directory Server backends are configured with the requested
backend ID userRoot
$ echo $?
1
