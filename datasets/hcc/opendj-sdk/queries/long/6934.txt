The https://ea.forgerock.com/docs/ds/install-guide/#available-profiles shows version numbers in the -set arguments, when in fact the version number is only valid in the -profile argument.

If you use a version number in a --set argument, you may get a message that the parameter is missing:

$ /path/to/opendj/setup \
>  --serverId evaluation-only \
>  --deploymentKeyPassword password \
>  --rootUserDN uid=admin \
>  --rootUserPassword password \
>  --hostname opendj.example.com \
>  --ldapPort 1389 \
>  --httpPort 8080 \
>  --adminConnectorPort 4444 \
>  --profile ds-app-data:7.0.0 \
>  --set ds-app-data:7.0.0/baseDn:o=ForgeRock.com \
>  --set ds-app-data:7.0.0/ldifFile:/path/to/proxyUser.ldif \
>  --acceptLicense
The parameter 'baseDn' is required by profile 'ds-app-data', but no value has
been provided
