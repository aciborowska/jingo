It appears that the LDAP keystore implementation no longer allows deletion of a value of the description attribute. This behavior differs from that of the JE backend. For example:

$ /home/mark/Repositories/opendj-docs/doc-tests/target/bats/opendj/bin/ldapsearch -p 38017 -D uid=admin -w password -b ds-keystore-alias=private-key,ou=keystore,dc=example,dc=com -s base \& description
dn: ds-keystore-alias=private-key,ou=keystore,dc=example,dc=com
description: LDAP keystore

$ /home/mark/Repositories/opendj-docs/doc-tests/target/bats/opendj/bin/ldapmodify -p 38017 -D uid=admin -w password 
dn: ds-keystore-alias=private-key,ou=keystore,dc=example,dc=com
changetype: modify
delete: description
description: LDAP keystore

# The LDAP modify request failed: 16 (No Such Attribute)
# Additional Information:  Entry ds-keystore-alias=private-key,ou=keystore,dc=example,dc=com cannot be modified because an attempt was made to remove one or more values from attribute description but this attribute is not present in the entry

$ /home/mark/Repositories/opendj-docs/doc-tests/target/bats/opendj/bin/ldapsearch -p 38017 -D uid=admin -w password -b uid=bjensen,ou=people,dc=example,dc=com -s base \& description
dn: uid=bjensen,ou=People,dc=example,dc=com
description: Original description

$ /home/mark/Repositories/opendj-docs/doc-tests/target/bats/opendj/bin/ldapmodify -p 38017 -D uid=admin -w password 
dn: uid=bjensen,ou=People,dc=example,dc=com
changetype: modify
delete: description
description: Original description

# MODIFY operation successful for DN uid=bjensen,ou=People,dc=example,dc=com
