Found using OpenDJ 3.0.0 rev

Scenario

	configure an instance

$ ./opendj/setup -h localhost -p 1389 -D "cn=myself" -w password --adminConnectorPort 4444 --ldapsPort 1636 --generateSelfSignedCertificate -b dc=example,dc=com -d 10 --cli -n --acceptLicense


	enable the HTTP connection handler

$ ./opendj/bin/dsconfig -h localhost -p 4444 -D "cn=myself" -w password -X set-connection-handler-prop --handler-name "HTTP Connection Handler" --set enabled:true --set listen-port:8080 -n


	enable the Common Audit Csv File HTTP Access Logger

$ ./opendj/bin/dsconfig -h  localhost -p 4444 -D "cn=myself" -w password -X set-log-publisher-prop --publisher-name "Common Audit Csv File HTTP Access Logger" --set enabled:true -n


	enable Common Audit Transaction ID

$ ./opendj/bin/dsconfig -h localhost -p 4444 -D "cn=myself" -w password -X set-global-configuration-prop --set trust-transaction-ids:true -n





After that we can see in the "opendj/logs" folder that we have 2 "http-access.csv" files
