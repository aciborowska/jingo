I found this issue with 7.0.0-SNAPSHOT rev. 18145b88f24 on Windows Server. 

When trying to create a backup with dsbackup tool I wasn't able to figure out how to specify path to folder where the backup will be stored.

The documentation mention URI but I tried several combinations and doesn't work for me. I also tried absolute path and that didn't work too (this works on linux).
Expected behavior

I expect that following is supported on Linux and Windows:

	support Windows file:/// URLs
	support Windows non-URLs like E:\some\path
