There seems to have been a regression in REST2LDAP for listing groups.

To reproduce, set up the server with the ds-evaluation profile, and try to list groups. For example:

$ curl  --cacert ca-cert.pem  --user kvaughan:bribery  https://opendj.example.com:8443/api/groups/?_queryFilter=true\&_prettyPrint=true
{
  "result" : [ {
    "_rev" : "00000000dd4d104e",
    "_schema" : "frapi:opendj:rest2ldap:group:1.0",
    "_meta" : { },
    "description" : "Groups that authenticated users can manage on their own"
  } ],
  "resultCount" : 1,
  "pagedResultsCookie" : null,
  "totalPagedResultsPolicy" : "NONE",
  "totalPagedResults" : -1,
  "remainingPagedResults" : -1
}

In fact there are several groups:

$ ldapsearch -p 1389 -h localhost -b ou=groups,dc=example,dc=com -D uid=admin -w password "(&)"  1.1
dn: ou=Groups,dc=example,dc=com

dn: cn=Accounting Managers,ou=groups,dc=example,dc=com

dn: cn=Directory Administrators,ou=Groups,dc=example,dc=com

dn: cn=HR Managers,ou=groups,dc=example,dc=com

dn: cn=PD Managers,ou=groups,dc=example,dc=com

dn: cn=QA Managers,ou=groups,dc=example,dc=com

dn: ou=Self Service,ou=Groups,dc=example,dc=com

dn: cn=Carpoolers,ou=Self Service,ou=Groups,dc=example,dc=com



The Carpoolers group is a groupOfNames, whereas the others are groupOfUniqueNames. But Carpoolers shouldn't show up in the query with the sample REST2LDAP config anyway.
