Explanation from email:
While setting up a proxy by following https://qa-backstage.forgerock.com/docs/ds/7/install-guide/chap-setup-proxy.html#test-proxy, I've run into an issue.
 I believe that import-ldif may not reference the right location of the proxyUser.ldif.

The original command is:

import-ldif \
 --hostname opendj.example.com \
 --port 14444 \
 --bindDN uid=admin \
 --bindPassword password \
 --backendId proxyUser \
 --ldifFile proxyUser.ldif \
 --usePkcs12TrustStore /path/to/opendj/config/keystore \
 --trustStorePasswordFile /path/to/opendj/config/keystore.pin \
 --no-prompt

And I think it should be:

import-ldif \
 --hostname opendj.example.com \
 --port 14444 \
 --bindDN uid=admin \
 --bindPassword password \
 --backendId proxyUser \
 --ldifFile db/proxyUser/proxyUser.ldif \
 --usePkcs12TrustStore /path/to/opendj/config/keystore \
 --trustStorePasswordFile /path/to/opendj/config/keystore.pin \
 --no-prompt

(see the difference in the --ldifFile option)
