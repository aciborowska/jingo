Changing some dsconfig properties require a server restart to take effect, but dsconfig does not warn about them, while it should.

The property value is recorded (doing a dsconfig get property value returns the correct new value), but the server does not use the new value until the restart.

Impacted properties :

	db-log-file-max from backend
	probably db-log-filecache-size from backend (need to be confirmed)




$ dsconfig -h localhost -p 4444 -D "cn=Directory Manager" -w password -X set-backend-prop --backend-name userRoot --set db-log-file-max:1gib

The equivalent non-interactive command-line is:
dsconfig set-backend-prop \
          --backend-name userRoot \
          --set db-log-file-max:1gib \
          --hostname localhost \
          --port 4444 \
          --bindDn cn=Directory\ Manager \
          --trustAll \
          --bindPassword ****** \
          --no-prompt
