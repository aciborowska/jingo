The problem was discovered when running ForgeRock OpenDJ Administration course's lab on replication.

From a previous lab, a custom password policy was defined and assigned to an entry 'uid=helpdeskAdmin,ou=Administrators,dc=example,dc=com' on server ds1.
In the replication lab, the server ds2 gets initialzed from ds1, but the password policy wasn't created (under cn=config).
ds2 server reports some error, discard the conflicting entry, but continues with the full import successfully.

It looks like some regression introduced with the refactoring for the Authentication Policy.
