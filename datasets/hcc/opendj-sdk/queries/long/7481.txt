The JSON File-Based Access Logger should include the proxy authorization DN (if set) in a "runAs" field.

However it appears that the object the logging code is querying for the proxy authz DN never has it set, so as a result the logger never logs the proxy authz DN.
STEPS TO REPRODUCE

Create a server, and load in this LDIF:

dn: dc=example,dc=com
objectclass: domain
dc: example

dn: ou=people,dc=example,dc=com
objectclass: organizationalUnit
ou: people
aci: (targetcontrol="2.16.840.1.113730.3.4.18")(version 3.0; acl "control"; allow(read) userdn="ldap:///uid=user.0,ou=people,dc=example,dc=com";)
aci: (targetattr="*")(version 3.0; acl "proxy r/w"; allow(proxy) userdn="ldap:///uid=user.0,ou=people,dc=example,dc=com";)

dn: uid=user.0,ou=people,dc=example,dc=com
objectclass: inetOrgPerson
cn: user
sn: user
uid: user.0
userPassword: password
ds-privilege-name: proxied-auth

dn: uid=user.1,ou=people,dc=example,dc=com
objectclass: inetOrgPerson
cn: user
sn: user
uid: user.1
userPassword: password
description: original value


Do an ldapmodify with proxy auth:


ldapmodify -h localhost -p 636 -Z -X -D "uid=user.0,ou=people,dc=example,dc=com" -w password -Y u:user.1 <<LDIF
dn: uid=user.1,ou=people,dc=example,dc=com
changetype: modify
replace: description
description: added with proxy auth
-

LDIF


Check the logs for "runAs".

You can also check that the server is performing the proxy auth correctly by checking the modifiersName in the modified entry: it should say user.1 and not user.0.
