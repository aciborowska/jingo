Invalid binary attribute value is returned when using Rest2LDAP.

Steps to reproduce the issue

1/ Enable HTTP Connection Handler on your server.
2/ Add a binary attribute like jpegPhoto to an entry.
eg. jpegphoto:<file:///tmp/Samantha-Carter.jpg

3/ Modify http-config.json config.
Add the following line to "attributes" : {


"jpegPhoto" : { "simple"   : { "ldapAttribute" : "jpegPhoto", "isSingleValued" : true, "isRequired" : true, "isBinary" : true } },



Restart server

4/ Compare jpegPhotoValue from ldif / json value

	ldif : check via control-panel or ldapsearch

 ./bin/ldapsearch -p 1389 -b "uid=user.1,ou=people,dc=example,dc=com" "objectclass=*"






	json : 

curl --user <login>:<password> http://<localhost>:<port>/users/user.1?_prettyPrint=true
