Example of messages with wrong severity in the errors log.

1. The following messages related to ACI validation (here this is just a subset, maybe all messages starting with "The provided Access Control Instruction (ACI)") should have severity "WARNING" instead of "ERROR":

[04/Nov/2014:23:29:32 +0000] category=SCHEMA severity=ERROR msgID=messages.access_control.7 msg=The provided Access Control Instruction (ACI) bind rule value "userattr=""" is invalid. A valid bind rule value must be in the following form: keyword operator "expression"
[04/Nov/2014:23:29:32 +0000] category=SCHEMA severity=ERROR msgID=messages.access_control.75 msg=The provided Access Control Instruction (ACI) expression value "ldap:///cn=Marketing,ou=Groups,o=ACI Tests,dc=example,dc=com" is invalid because it contains the roledn keyword, which is not supported, replace it with the groupdn keyword
[04/Nov/2014:23:33:26 +0000] category=SCHEMA severity=ERROR msgID=messages.access_control.53 msg=The provided Access Control Instruction (ACI) targattrfilter expression value addddddd=title:(title=Eng*) is invalid because it is not in the correct format.A valid targattrsfilters expression value must be in the following format: "add=attr1: F1 && attr2: F2 ... && attrN: FN,del= attr1: F1 && attr2: F2 ... && attrN: FN" 



2. The following messages related to import should have severiry "NOTICE" instead of "ERROR":

[05/Nov/2014:00:19:25 +0000] category=JEB severity=ERROR msgID=messages.jeb.178 msg=DN phase two processing completed. Processed 164 DNs



3. The following messages related to replication should have severity "NOTICE" instead of "ERROR":

[05/Nov/2014:00:26:14 +0000] category=SYNC severity=ERROR msgID=messages.replication.142 msg=Finished total update: imported domain "cn=schema" from remote directory server DS(22447) to this directory server DS(17669).
