When trying to use Apache Directory Studio to connect to ForgeRock DS or ForgeRock Proxy, as a user different from uid=Admin, an error is reported with the following message:


"missing schema location in rootdse using default schema"



It turns out that the subSchemaSubEntry attribute is not in the list of attributes that can be read (anonymously, or authenticated) from the rootDSE, in global-aci (DS), or global access control policy (Proxy).

Adding the attribute removes the error.
On the proxy side, it consist of adding the following line to the `dn: cn=Root DSE access,cn=Global Access Control Policies,cn=Access Control Handler,cn=config` entry:


ds-cfg-allowed-attribute: subSchemaSubEntry



On the DS side, the attribute must be added to this global-aci:



ds-cfg-global-aci: (target="ldap:///")(targetscope="base") (targetattr="objectClass||namingContexts||supportedAuthPasswordSchemes||supportedControl||supportedExtension||supportedFeatures||supportedLDAPVersion||supportedSASLMechanisms||supportedTLSCiphers||supportedTLSProtocols||vendorName||vendorVersion||fullVendorVersion||alive||healthy")(version 3.0; acl "User-Visible Root DSE Operational Attributes"; allow (read,search,compare) userdn="ldap:///anyone";)
