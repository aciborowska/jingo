Found while aligning doc tests with code, I notice that a long character set, such as the one now used by default for the random password generator, now folds in the middle when output with dsconfig:

Property               : Value(s)
-----------------------:-------------------------------------------------------
enabled                : true
password-character-set : alphanum:abcdefghijklmnopqrstuvwxyzABCDEFGHIJKLMNOPQRS
                       : TUVWXYZ0123456789
password-format        : alphanum:10 

If you copy alphanum to edit it, you'll have to manually remove the newline, spaces and colon.

Previously, the output for the character set was all on one line:

password-character-set : alphanum:abcdefghijklmnopqrstuvwxyzABCDEFGHIJKLMNOPQRSTUVWXYZ0123456789
