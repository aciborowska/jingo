Discovered with daily tests and it's new issue somewhere between commit (84e7df93a44)  and commit (5d6b47e05e2)

To reproduce the issue:

./run-pybot.py -s groups_group.staticmember -t Member_Delete_All_Attributes_In_A_Static_Group_Entry -v opendj




Lets have an entry which is member of some groups:


dn: uid=user_01,ou=static,dc=groups,dc=com 
isMemberOf: cn=my_staticgroup1,dc=groups,dc=com
isMemberOf: cn=my_staticgroup2,dc=groups,dc=com 
isMemberOf: cn=my_staticgroup3,dc=groups,dc=com



Steps:


	Create static group:


dn:cn=my_newStaticGroup,dc=groups,dc=com 
cn:my_newStaticGroup 
objectclass:top 
objectclass:groupofnames 
description:this is the description of my_newStaticGroup
member:uid=user_01,ou=static,dc=groups,dc=com
member:uid=user_02,ou=static,dc=groups,dc=com
member:uid=user_11,ou=static,dc=groups,dc=com


	Delete member attribute of my_newStaticGroup


dn: cn=my_newStaticGroup,dc=groups,dc=com 
changetype: modify 
delete: member 
member: uid=user_01,ou=static,dc=groups,dc=com 
member: uid=user_02,ou=static,dc=groups,dc=com
member: uid=user_11,ou=static,dc=groups,dc=com


	Check that entry `uid=user_01,ou=static,dc=groups,dc=com` is not member of above group, but it is member of other groups he belongs to:


./bin/ldapsearch -h pyforge.example.com -p 1389 -D "cn=Directory Manager" -w password -b "uid=user_01,ou=static,dc=groups,dc=com" "(objectclass=*)" isMemberOf

STDOUT:
dn: uid=user_01,ou=static,dc=groups,dc=com

It used to be:


dn: uid=user_01,ou=static,dc=groups,dc=com 
isMemberOf: cn=my_staticgroup1,dc=groups,dc=com
isMemberOf: cn=my_staticgroup2,dc=groups,dc=com 
isMemberOf: cn=my_staticgroup3,dc=groups,dc=com
