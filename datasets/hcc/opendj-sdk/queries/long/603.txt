On Linux, create-rc-script uses incorrect script for "su"

$ create-rc-script -u ldap -f /tmp/opendj-rc-script

generates the following lines:

  /bin/su ldap "${INSTALL_ROOT}/bin/start-ds" --quiet
  /bin/su ldap "${INSTALL_ROOT}/bin/stop-ds" --quiet
  /bin/su ldap "${INSTALL_ROOT}/bin/stop-ds" --restart --quiet

Such invocation generates an error because --quiet is passed to su, which is invalid for that command.

It really should be

  /bin/su ldap --command "${INSTALL_ROOT}/bin/start-ds --quiet"
