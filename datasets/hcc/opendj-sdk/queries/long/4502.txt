The procedure for setting the group-id on an RS was changed in 93275304d6 from this:


$ dsconfig \
 set-replication-server-prop \
 --port 4444 \
 --hostname rs.example.com \
 --bindDN "cn=Directory Manager" \
 --bindPassword password \
 --provider-name "Multimaster Synchronization" \
 --set group-id:1 \
 --trustAll \
 --no-prompt


to this (in replication.bash):


$ dsconfig \
  set-replication-domain-prop \
  --hostname ${hostname2} \
  --port ${adminPort2} \
  --bindDN "cn=Directory Manager" \
  --bindPassword password \
  --provider-name "Multimaster Synchronization" \
  --domain-name "dc=example,dc=com" \
  --set group-id:2 \
  --trustAll \
  --no-prompt


Trying to execute the set-replication-domain-prop version fails with:


Unable to continue since there are no "dc=example,dc=com" currently configured
on the server
