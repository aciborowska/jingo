A single DS instance can have several replicas, one for each domain name: for cn=admin data, cn=schema and for the user root (dc=example,dc=com is the usual examples for OpenDJ).
In the Prometheus endpoint, replica metrics must be differentiated using labels. We are currently missing one label for the domain name of the replica.

The affected metrics are:

	ds_replication_replica_assured_sd_acknowledged_updates
	ds_replication_replica_assured_sd_sent_updates
