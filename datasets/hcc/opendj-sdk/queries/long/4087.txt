On DJ 3.0, 3.5.1 and 4.0.0 (DS 5), persistent search on cn=changelog returns search results without any Entry Change Notification control.
 The control was returned in DJ 2.6.4.

On DJ 4.0.0:



./ldapsearch -p 5389 --persistentSearch ps:any -b "cn=changelog" -D "cn=Directory Manager" -w password "objectclass=*"

dn: changeNumber=5012,cn=changelog
 ...



On DJ 2.6.4:



./ldapsearch -p 2389 --persistentSearch ps:any -b "cn=changelog" -D "cn=Directory Manager" -w password "objectclass=*"

dn: cn=changelog
 objectClass: top
 objectClass: container
 cn: changelog

Persistent search change type: add <---------
 dn: changeNumber=3,cn=changelog
 ...
