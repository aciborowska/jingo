Similar to OPENDJ-1976, but with embedded OpenDJ setup as part of OpenAM install.

Reproduction steps, tested with OpenAM 13.5:

Check 8dot3name is disabled, i.e. `fsutil 8dot3name query c:` (can be disabled with `fsutil behavior set disable8dot3 1`). 

Microsoft has disabled 8dot3 format on all volumes other than C: by default.

Run OpenAM setup
Set OpenAM home directory with a space character, i.e. "C:\programs\openam home\"
Choose embedded OpenDJ and default options

Configuration will hang with the following:




Checking license acceptance...License terms accepted.
Checking configuration directory C:/openam home....Success.
Installing OpenAM configuration store...Success RSA/ECB/OAEPWithSHA1AndMGF1Padding.
Extracting OpenDJ, please wait...Complete
Running OpenDJ setupSetup command: --cli --adminConnectorPort 4444 --baseDN dc=openam,dc=forgerock,dc=org --rootUserDN cn=Directory Manager --ldapPort 50389 --skipPortCheck --rootUserPassword xxxxxxx --jmxPort 1689 --no-prompt --doNotStart --hostname openam.example.com --noPropertiesFile --backendType je Configuring Directory Server ..... 




No error is given.

Observations:

Enabling 8dot3 naming allows the setup to complete (`fsutil behavior set disable8dot3 0`)

Embedded DJ setup will also hang in the same way if the OpenAM home directory contains space characters in the top level directory i.e. `C:\openam home\`, regardless of 8dot3 naming setting.

Expectation:
OpenAM/DJ setup should be able to complete without 8dot3 naming.
