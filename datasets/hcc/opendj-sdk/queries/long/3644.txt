I'm trying to evaluate the effect of the last-login-time-format attribute. I set the attribute to "YYYYMMddHHmmssZ" with dsconfig and do some testing.

I then set the attribute to "yyyyMMddHHmmssZ" using dsconfig:


$ bin/dsconfig -D "cn=Directory Manager" -w password set-password-policy-prop --policy-name Default\ Password\ Policy --set last-login-time-format:yyyyMMddHHmmssZ --no-prompt --trustAll



No errors are returned, yet if I check the property again I see the previous value. (Confirmed by single stepping through the server's setLastLoginTime() method, and confirmed by checking the raw config.ldif file.)

This only happens if the two values only differ by case. Because the format specification used by java.text.SimpleDateFormat is case-sensitive, varying the value just by case seems to be a likely thing to do.

The definition of the ds-cfg-last-login-time-format attribute uses EQUALITY caseIgnoreMatch, which surely should be EQUALITY caseExactMatch?
