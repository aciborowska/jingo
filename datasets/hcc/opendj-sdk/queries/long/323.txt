If you attempt to rebuild an index that doesn't exist, while OpenDJ is running, then the task completes with errors but the backend is left offline.

$ rebuild-index -baseDN dc=example,dc=com -bindDN cn=Directory\ Manager -bindPassword password --port 4444 --index getpwent --start 0 --trustAll

Access log contains:-

[18/Oct/2011:16:36:04 +0100] category=BACKEND severity=NOTICE msgID=9896349 msg=Rebuild Index task 20111018163601003 started execution
[18/Oct/2011:16:36:04 +0100] category=BACKEND severity=NOTICE msgID=9896306 msg=The backend userRoot is now taken offline
[18/Oct/2011:16:36:04 +0100] category=TOOLS severity=SEVERE_ERROR msgID=10748756 msg=An error occurred while attempting to perform index rebuild:  There is no index configured for attribute type 'getpwent'
[18/Oct/2011:16:36:04 +0100] category=BACKEND severity=NOTICE msgID=9896350 msg=Rebuild Index task 20111018163601003 finished execution
