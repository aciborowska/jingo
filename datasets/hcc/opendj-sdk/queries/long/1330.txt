Found on the trunk (revision 10240).

Some replication monitoring information (cn=monitor) are no more available after a change in the replication topology.
Need to restart the instance to have the monitoring information available again.

Scenario:
*********
1. install a first instance using the GUI
=> in the topology panel, select "This server will be part of a replication topology"

2. search for replication topology information under cn=monitor on the first instance

$ ./opendj/bin/ldapsearch -h localhost -D "cn=directory manager" -w password -b "cn=dc_example_dc_com,cn=Replication,cn=monitor" -p 1389 -T "objectclass=*" dn
dn: cn=dc_example_dc_com,cn=Replication,cn=monitor

dn: cn=Directory server DS(27904) localhost:40253,cn=dc_example_dc_com,cn=Replication,cn=monitor

dn: cn=Replication server RS(14169) myhost:8989,cn=dc_example_dc_com,cn=Replication,cn=monitor

dn: cn=Connected directory server DS(27904) localhost:40253,cn=Replication server RS(14169) myhost:8989,cn=dc_example_dc_com,cn=Replication,cn=monitor



=> the entry "cn=Directory server DS(...)" exist as expected

3. install a second instance using the GUI and enable the replication with the first instance

4. search for replication topology information under cn=monitor on the first instance

$ ./opendj/bin/ldapsearch -h localhost -D "cn=directory manager" -w password -b "cn=dc_example_dc_com,cn=Replication,cn=monitor" -p 1389 -T "objectclass=*" dn
dn: cn=dc_example_dc_com,cn=Replication,cn=monitor

dn: cn=Replication server RS(14169) myhost:8989,cn=dc_example_dc_com,cn=Replication,cn=monitor

dn: cn=Connected directory server DS(27904) localhost:40331,cn=Replication server RS(14169) myhost:8989,cn=dc_example_dc_com,cn=Replication,cn=monitor

dn: cn=Connected replication server RS(2064) myhost:9989,cn=Replication server RS(14169) myhost:8989,cn=dc_example_dc_com,cn=Replication,cn=monitor

dn: cn=Connected directory server DS(21850) localhost:38897,cn=Connected replication server RS(2064) myhost:9989,cn=Replication server RS(14169) myhost:8989,cn=dc_example_dc_com,cn=Replication,cn=monitor



=> the entry "cn=Directory server DS(...)" is no move available

5. restart the first instance

6. search for replication topology information under cn=monitor on the first instance

./opendj/bin/ldapsearch -h localhost -D "cn=directory manager" -w password -b "cn=dc_example_dc_com,cn=Replication,cn=monitor" -p 1389 -T "objectclass=*" dn
dn: cn=dc_example_dc_com,cn=Replication,cn=monitor

dn: cn=Directory server DS(27904) localhost:40440,cn=dc_example_dc_com,cn=Replication,cn=monitor

dn: cn=Replication server RS(14169) myhost:8989,cn=dc_example_dc_com,cn=Replication,cn=monitor

dn: cn=Connected directory server DS(27904) localhost:40440,cn=Replication server RS(14169) myhost:8989,cn=dc_example_dc_com,cn=Replication,cn=monitor

dn: cn=Connected replication server RS(2064) myhost:9989,cn=Replication server RS(14169) myhost:8989,cn=dc_example_dc_com,cn=Replication,cn=monitor

dn: cn=Connected directory server DS(21850) localhost:38897,cn=Connected replication server RS(2064) myhost:9989,cn=Replication server RS(14169) myhost:8989,cn=dc_example_dc_com,cn=Replication,cn=monitor



=> the entry "cn=Directory server DS(...)" is now available again
