The property response-result-code-equal-to does not work in the right order.

For example:



	Provide basic ldapsearch command on existing user
	
		In log appears message with result code 0.
	
	




	Set filtering criteria for response-result-code-equal-to as below:
	
		dsconfig -h localhost -p 4444 -D 'cn=Directory Manager' --no-prompt -w password set-log-publisher-prop --publisher-name "File-Based Access Logger" --set filtering-policy:inclusive
		dsconfig -h localhost -p 4444 -D 'cn=Directory Manager' --no-prompt -w password set-access-log-filtering-criteria-prop --publisher-name "File-Based Access Logger" --criteria-name "Log only messages with result code equal to 0" --set response-result-code-equal-to:0
	
	




	Provide basic ldapsearch command on existing user
	
		In log does not appear message with result code 0.
	
	



NOTE: Also tested the opposite property 'response-result-code-not-equal-to:'   and works fine.
