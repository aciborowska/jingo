In the "Administration Guide":

	Chapter 4. Configuring Connection Handlers
	4.7. RESTful Client Access
	4. If necessary, change the connection handler configuration using the dsconfig command.

$ dsconfig
 set-connection-handler-prop
 --hostname opendj.example.com
 --port 4444
 --bindDN "cn=Directory Manager"
 --bindPassword password
 --handler-name "HTTP Connection Handler"
 --set listen-port:8443
 --set use-ssl:true
 --set key-manager-provider:JKS
 --set trust-manager-provider:"Blind Trust"
 --no-prompt





==> we should add the "--trustAll" option in this command


And in this example we use the "Blind Trust" Trust Manager Provider but by default this provider is not enabled.
So when we try to use this command we get the following message:
The HTTP Connection Handler could not be modified because of the following
reason:


	The Trust Manager Provider "Blind Trust" referenced in property
       "trust-manager-provider" is disabled



We should add at the beginning the command to enable the "Blind Trust" Trust Manager Provider:


$ dsconfig
 set-trust-manager-provider-prop
 --hostname opendj.example.com
 --port 4444
 --bindDN "cn=Directory Manager"
 --bindPassword password
 --provider-name "Blind Trust"
 --set enabled:true
 --no-prompt
 --trustAll
