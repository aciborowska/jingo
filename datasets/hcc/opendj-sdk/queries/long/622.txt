An extendedRequest containing a plain requestValue doesn't get sent correctly to the LDAP server. For example:


<extendedRequest>
    <requestName>2.7.1.8.2.8</requestName>
    <requestValue>hello world</requestValue>
</extendedRequest>



The LDAP server gets a requestValue string like "[requestValue: null]".

Using a base-64 encoded value works:


<extendedRequest>
    <requestName>2.7.1.8.2.8</requestName>
    <requestValue xsi:type="xsd:base64Binary">Rm9yZ2VSb2Nr</requestValue>
</extendedRequest>



In this case the LDAP servers get a requestValue string of "ForgeRock".
