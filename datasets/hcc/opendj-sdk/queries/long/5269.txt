In some DSML tests we send invalid SOAP request (invalid XML) and we expect to get a proper message but we now get AttributeError errors.
1. request with invalid XML
Expected error:

    <batchResponse xmlns="urn:oasis:names:tc:DSML:2:0:core">
      <errorResponse type="malformedRequest">
        <message>javax.xml.transform.TransformerException: org.xml.sax.SAXParseException: Premature end of file.</message>
      </errorResponse>
    </batchResponse>



Got:

UnicodeDecodeError: 'utf-8' codec can't decode byte 0xea in position 132: invalid continuation byte



To reproduce the issue:

./run-pybot.py -s dsml_group.03error -t soap_garbage DJ



2. request with errorneous SOAP namespace
Expected error:

    <batchResponse xmlns="urn:oasis:names:tc:DSML:2:0:core" requestID="SOAP namespace is erroneous">
      <errorResponse type="malformedRequest">
        <message>null</message>
      </errorResponse>
    </batchResponse>



Got:

AttributeError: no such child: {urn:oasis:names:tc:DSML:2:0:core}message



To reproduce the issue:

./run-pybot.py -s dsml_group.03error -t soap_nosoapnamespace3 DJ
