It looks like adding an entry with binary attribute options fails during reading of ldif. Looks like a recent regression in 2.5.0 at least since r7881. Works on 2.4.5 

Examples:

bin/ldapmodify -a -p 1389 -D "cn=Directory Manager" -w secret12 < /tmp/certuser.ldif 
Error at or near line 1 in LDIF file Console:
org.opends.server.util.LDIFException: Unable to parse LDIF entry cn=Jeff
Grapes,dc=example,dc=com starting at line 1 because it has an invalid binary
option for attribute userCertificate

cat /tmp/certuser.ldif
dn: cn=Jeff Grapes, dc=example,dc=com
objectClass: top
objectClass: person
objectclass: strongauthenticationUser
cn: Jeff Grapes
sn: Grapes
userCertificate;binary:< file:///tmp/user-cert.pem

bin/ldapmodify -a -p 1389 -D "cn=Directory Manager" -w secret12 < /tmp/certuser2.ldif 
Error at or near line 1 in LDIF file Console:
org.opends.server.util.LDIFException: Unable to parse LDIF entry cn=Jeff
Grapes,dc=example,dc=com starting at line 1 because it has an invalid binary
option for attribute userCertificate

cat /tmp/certuser2.ldif
dn: cn=Jeff Grapes, dc=example,dc=com
objectClass: top
objectClass: person
objectclass: strongauthenticationUser
cn: Jeff Grapes
sn: Grapes
