When running ldifSearch against a file, I noticed that operational attributes such as createTimestamp are output by default even if not requested (or just a specific attribute is requested).

Only requested attributes should be returned, and operational ones should only be returned if explicitly requested.

$ /Space/Tests/DS6/opendj/bin/ldifsearch -b cn=Index,ds-cfg-backend-id=userRoot,cn=Backends,cn=config -s one config.ldif '(&)' dn

dn: ds-cfg-attribute=accountno,cn=Index,ds-cfg-backend-id=userRoot,cn=Backends,cn=config
creatorsName: cn=Directory Manager,cn=Root DNs,cn=config
entryUUID: 894e4b99-5f73-4800-a854-791cb93f3cec
createTimestamp: 20180328034229Z

dn: ds-cfg-attribute=aci,cn=Index,ds-cfg-backend-id=userRoot,cn=Backends,cn=config
