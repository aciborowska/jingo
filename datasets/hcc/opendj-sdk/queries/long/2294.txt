I did not test, but I am pretty sure this also affects 2.6.

Scenario

Setup


# Setup and start OpenDJ 

# Set the error log account status notification handler for the default password policy
target/package/opendj_auto/bin/dsconfig --hostname localhost -p 4444 -D "cn=Directory Manager" -w admin -X   set-password-policy-prop --policy-name "Default Password Policy" --set account-status-notification-handler:"Error Log Handler" --no-prompt



Test 1: disable account



target/package/opendj_auto/bin/ldapmodify -p 1389 -D "cn=Directory Manager" -w admin <<END_OF_COMMAND_INPUT
dn: uid=user.0,ou=People,dc=example,dc=com
changetype: modify
replace: ds-pwp-account-disabled
ds-pwp-account-disabled: true
END_OF_COMMAND_INPUT



Expected result


	New line in logs/errors:

[21/Sep/2015:09:57:20 +0200] category=EXTENSIONS severity=NOTICE msgID=org.opends.messages.extension.375 msg=Account-Status-Notification type='account-enabled' userdn='uid=user.0,ou=People,dc=example,dc=com' id=553 msg='The user account has been administratively enabled'





Actual result

No output

Test 2: enable account



target/package/opendj_auto/bin/ldapmodify -p 1389 -D "cn=Directory Manager" -w admin <<END_OF_COMMAND_INPUT
dn: uid=user.0,ou=People,dc=example,dc=com
changetype: modify
replace: ds-pwp-account-disabled
ds-pwp-account-disabled: false
END_OF_COMMAND_INPUT



Expected result


	New line in logs/errors:

[21/Sep/2015:09:57:31 +0200] category=EXTENSIONS severity=NOTICE msgID=org.opends.messages.extension.375 msg=Account-Status-Notification type='account-disabled' userdn='uid=user.0,ou=People,dc=example,dc=com' id=554 msg='The user account has been administratively disabled'





Actual result

No output
