Since OPENDJ-3527, all tools which create tasks must specify the --offline parameter to not start a task in a running server.
It seems that the embedded api for rebuild-index and import-ldif tools is starting these tools in offline mode since no connections parameters are provided in RebuildIndexParameters#toCommandLineArguments() and ImportParameters.toCommandLineArguments().
We need to add the offline parameter to these command line argument to fix this issue.
