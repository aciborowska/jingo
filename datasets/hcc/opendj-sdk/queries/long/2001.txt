After investigating this a bit further it appears that this test failure happens because the test is unable to cope with the fact that my locale uses decimal comma instead of decimal point.
Considering that the command output uses the comma characters to separate the different columns, the proposed solution is to ensure that the tool always uses decimal points for printing out the statistics. This should also make the tool output more reliable, and allow easier processing.
