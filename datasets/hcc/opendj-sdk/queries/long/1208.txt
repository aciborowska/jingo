unzip opendj

use a properties file which includes 'rootUserPasswordFile' as suggested by http://docs.forgerock.org/en/opendj/2.6.0/install-guide/index.html#chap-install-cli

e.g.


hostname                        =config.doit.org
ldapPort                        =7389
ldapsPort                       =7636
adminConnectorPort              =7390
jmxPort                         =7391
rootUserDN                      =cn=Directory Manager
rootUserPasswordFile            =/var/tmp/ds.pwd 
baseDN                          =dc=openam,dc=forgerock,dc=org
addBaseEntry                    =true
generateSelfSignedCertificate   =true
enableStartTLS                  =false



run setup with properties file ... 


setup --cli --acceptLicense --no-prompt --propertiesFilePath /var/tmp/opendj-silentInstall.txt



This results in ...


An error occurred while parsing the command-line arguments:  An unexpected error occurred while attempting to initialize the command-line arguments:  ERROR:  No password was provided for the initial root user.  When performing a non-interactive installation, this must be provided using either the rootUserPassword or the rootUserPasswordFile argument



However running setup with option '--rootUserPasswordFile' works fine.
