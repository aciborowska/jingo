To reproduce on 6.5:

	Create a properties file with the contents:

ldapsearch.hostname=localhost
ldapsearch.port=1389
ldapsearch.bindDN=cn=Directory Manager
ldapsearch.bindPassword=password






	Perform an ldap operation, for example:

$ bin/ldapsearch --propertiesFilePath ldif/test.properties -b dc=example,dc=com "(objectClass=*)"

The LDAP bind request failed: 49 (Invalid Credentials)


According to the access log:

[07/Dec/2018:17:17:53 +0000] BIND RES conn=15 op=0 msgID=1 result=49 authFailureReason="Invalid Credentials: Unable to bind to the Directory Server using simple authentication because that user does not have a password" authDN="" etime=2





Note that the above works fine on DS 6.0.
