If the rest2ldap.war is deployed in a packed form, the lookup of the configuration directory fails with


java.lang.IllegalArgumentException: URI scheme is not "file"
	java.io.File.<init>(File.java:421)
	org.forgerock.opendj.rest2ldap.Rest2LdapHttpApplication.<init>(Rest2LdapHttpApplication.java:179)



Reason:
the 

URL configUrl = getClass().getResource("/config.json")

is a vfs:// URL (on JBoss EAP) and toURI() won't work on these URLs.
You can alternatively use 


this.configDirectory = configUrl != null ? new File(configUrl.getFile()).getParentFile() : null;



for better usability if added the possibility to configure the path from a system property.
Patch for both fixes is attached.
