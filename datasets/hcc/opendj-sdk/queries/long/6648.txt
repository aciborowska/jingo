Import-ldif behavior for option --ldifFile is not corresponding with documentation, where is used ldif file from the currently working directory.  It's using relative path to where the server is installed.
For below steps '--ldifFile db/test/data.ldif'. User has to guess that non-absolute paths are relative to where the server is installed, rather than what the current working directory is.

Steps to reproduce:

	can reproduce when try example from https://qa-backstage.forgerock.com/docs/ds/7/install-guide/chap-setup-proxy.html#test-proxy


	Create Testing folder, navigate to the folder, there unzip opendj 7

mkdir Testing
cd Testing
unzip /path/to/archive/opendj-7.0.0-snapshot

	Setup DJ

./opendj/setup   --deploymentKey $DEPLOYMENT_KEY   --deploymentKeyPassword $DEPLOYMENT_PASSWORD   --hostname localhost   --ldapPort 1389   --enableStartTls   --ldapsPort 1636   --adminConnectorPort 4444   --rootUserDN uid=admin   --rootUserPassword password   --profile ds-evaluation   --acceptLicense --quiet

	Create data.ldif file and insert data  data.ldif

touch data.ldif

Data

dn: o=ForgeRock.com
objectClass: top
objectClass: organization
o: ForgeRock.com

dn: cn=DS,o=ForgeRock.com
cn: DS
objectClass: top
objectClass: namedObject
objectClass: ds-certificate-user
ds-certificate-subject-dn: CN=DS, O=ForgeRock.com
ds-privilege-name: config-read
ds-privilege-name: proxied-auth


	Create test folder at opendj/db and there copy file data.ldif

mkdir opendj/db/test && cp data.ldif opendj/db/test

	Create backend test

./opendj/bin/dsconfig  create-backend  --hostname localhost  --port 4444  --bindDN uid=admin  --bindPassword password  --backend-name test  --type ldif  --set enabled:true  --set base-dn:o=ForgeRock.com  --set ldif-file:db/test/data.ldif  --usePkcs12TrustStore opendj/config/keystore  --trustStorePasswordFile opendj/config/keystore.pin --no-prompt

	Try import data.ldif from current working directory
./opendj/bin/import-ldif  --hostname localhost  --port 4444  --bindDN uid=admin  --bindPassword password  --backendId test  --ldifFile data.ldif  --usePkcs12TrustStore opendj/config/keystore  --trustStorePasswordFile opendj/config/keystore.pin  --no-prompt
