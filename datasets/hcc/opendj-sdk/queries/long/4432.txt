How to reproduce:

Use a tool which supports --propertiesFilePath and --noPropertiesFile options:


$ bin/ldapsearch --propertiesFilePath /tmp/test.properties --noPropertiesFile -h localhost -Z -p 1636 -D 'cn=directory manager' -w password -b dc=example,dc=com -X "&"
dn: dc=example,dc=com
objectClass: top
objectClass: domain
dc: example



Expected behavior:
The tool should not work and prints that these options are mutually exclusive

Actual behavior:
See above, the tool works fine, as a side note in our current parsing implementation, --noPropertiesFile takes precedence over --propertiesFilePath
