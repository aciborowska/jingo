On Windows build 'opendj-server-example-plugin' and compare the target jar's contents with one built on Linux/MacOS. Some properties files are missing, such as:
config/messages/com/example/opendj/meta/ExamplePluginCfgDefn.properties
config/profiles/cli/com/example/opendj/meta/ExamplePluginCfgDefn.properties
config/profiles/ldap/com/example/opendj/meta/ExamplePluginCfgDefn.properties

If built with Maven's '-X' setting, it's possible to see a mix of forward and back slashes in paths. E.g.



[INFO] Adding resource "C:\tempo\opendj-server-example-plugin\target/generated-resources/config/**/*.properties to resource path...



Apparently due to opendj-maven-plugin not making use of Java's File.separator.



 private Resource getGeneratedMavenResources() {
        final String[] generatedResourcesRelativePath =
                new String[] { "/META-INF/services/**", "/config/**/*.properties" };



It's expected the target jar of opendj-server-example-plugin will contain the same files regardless of OS used as a build platform.
