Proxy-Auth of Rootdn as the authzID gives the bindDN entry bypass-acl access and as a result, the bindDN can now bypass all DIT and Global ACI's.

Application ID = uid=appacct,ou=Service Accounts,dc=forgerock,dc=com

Searching Testcase: with uid=appacct as the bindDN:

This DIT has ACI's that do not allow uid=appacct to return more than 5 entries.

a) authzID as basic user = 0 nentries
b) authzID as the uid=appacct = 5 nentries
c) authzID as cn=Directory Manager = 21k nentries

Note: The testcase's new instance has 100 entries.

1) Using a default new user, proxy auth as the rootdn - no proxy-auth privilege.

:bin/$ ./ldapsearch -h localhost -p 1389 -D "uid=appacct,ou=Service Accounts,dc=forgerock,dc=com" -w password -Y "dn:cn=Directory Manager" -b ou=People,dc=forgerock,dc=com uid=user.*
SEARCH operation failed
Result Code:  123 (Authorization Denied)
Additional Information:  You do not have sufficient privileges to use the proxied authorization control

2) Add the proxy-auth privilege

:bin/$ ./ldapmodify -p 1389 -h localhost -D "cn=Directory Manager" -w password --filename ./mod

changetype: modify
add: ds-privilege-name
ds-privilege-name: proxied-auth

3) Search again
