Found using functional tests. This issue cause a lot of failed tests, so marking as impediment.

To reproduce the issue run following command:

 


./run-pybot.py -s clu_group.ldapmodify_DJ -t change_attribute_value -v opendj

 

 

The problem is that value of attribute is not changed by modification operation.

Steps:

	Setup simple DS with some entries
	Search entry and check value of postalCode attribute
	
		

./ldapsearch -h localhost -p 1389 -D "cn=Directory Manager" -w password -b "uid=user.1,ou=ldapmodify,o=clu tests,dc=example,dc=com"
...
postalCode: 93520
...

	
	




	Change attribute value of postalCode by ldapmodify
	
		

./ldapmodify -h localhost -p 1389 -D "cn=Directory Manager" -w password	
dn: uid=user.1,ou=ldapmodify,o=clu tests,dc=example,dc=com
changetype: modify 
replace: postalCode 
postalCode: 42421


	
	




	Search for the same entry and check the attribute value
	
		

./ldapsearch -h localhost -p 1389 -D "cn=Directory Manager" -w password -b "uid=user.1,ou=ldapmodify,o=clu tests,dc=example,dc=com"
...
