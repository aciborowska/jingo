Here is the scenario:
1. standardUser (non-privileged user) do an import or export
=> operation should fail due to insufficient access rights

2. adminUser adds "ldif-import"/"ldif-export" privilege to standardUser

$ ./ldapmodify ...
dn: standardUserDN
changetype: add
ds-privilege-name: ldif-import
ds-privilege-name: ldif-export



3. standardUser do an import or export
=> operation should fail due to insufficient access rights

4. adminUser adds the following global aci

ds-cfg-global-aci: target="ldap:///cn=Scheduled Tasks,cn=Tasks")(targetattr="ds-task-class-name || ds-task-export-backend-id || ds-task-export-ldif-file")(version 3.0; acl "Allows writes for tasks"; allow (write,add,delete) userdn="ldap:///anyone";)"



5. standardUser do an import or export
=> operation succeeds


In steps 1 and 3, the operation fails but with an unexpected error message:

SUCCESS: RC=1, Expected 1, Result=[[0, 'ERROR: An error occurred while trying to decode the response from the server:
null
']]



we should have:

SUCCESS: RC=1, Expected 1, Result=[[0, 'ERROR: An error occurred while trying to decode the response from the server:
The entry ds-task-id=20130617060335447,cn=Scheduled Tasks,cn=Tasks cannot be
added due to insufficient access rights
']]
