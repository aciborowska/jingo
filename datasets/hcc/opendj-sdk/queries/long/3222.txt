The NPE is trivial to understand in the source code.

According to the RFC-3829:
The authorization identity request control MAY be included in any bind request which specifies protocol version 3
We should decide how to fix this issue:

	Add a check on the client tools to return a nice error message if the --reportAuthzID option is used without any authentication parameters
	Do nothing and let the tool run as expected (behavior of server tools)
	Add a # Bound as anonymous information in stdout
