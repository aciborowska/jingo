In adapting doc samples for the patch for OPENDJ-286, I find that ds-pwp-last-login-time-attribute defaults to last-login-time.

Unfortunately there is no such attribute defined, and tests end up failing with an object class violation when a user subject to the policy performs a bind, e.g.:
