To reproduce this
1) create a new user using the Manage Entries GUI using "Entries->New User" and create a user where the Naming Attribute contains a \.
This is the LDIF view of the user I created:
dn: cn=domainuser,dc=my,dc=org
cn: domain\user
givenName: Test
objectClass: organizationalPerson
objectClass: top
objectClass: person
objectClass: inetOrgPerson
sn: User

At this point, the new entry displays fine.  However, close the Manage Entries window and load it again and if you try to load the entry the following error is displayed:
Error searching entry  An error occurred searching entry 'cn=domain\\\user,dc=my,dc=org'. Details: java.lang.IllegalArgumentException: Not a valid attribute string value:domain\\\user,improper usage of backslash
