When running the setup in interactive mode, we have the default value for the "populate the database" option:

Options for populating the database:

    1)  Only create the base entry
    2)  Leave the database empty
    3)  Import data from an LDIF file
    4)  Load automatically-generated sample data

Enter choice [1]: 



==> so by default "Only create the base entry"

But when running the setup in non-interactive mode:

$ ./opendj/setup --cli -p 1389 --adminConnectorPort 4444 -D cn=myself -w password -b "dc=com" -n



==> the instance is configured with  an empty database

So we need to make 'leave database empty' option default in interactive mode to be consistent.
