Our debug logging does not behave very predictably. With this debug target:



dn: ds-cfg-debug-scope=org.opends.server.core.SearchOperation#processSearch,cn=Debug Targets,cn=File-Based Debug Logger,cn=Loggers,cn=config
objectClass: top
objectClass: ds-cfg-debug-target
ds-cfg-debug-exceptions-only: true
ds-cfg-debug-scope: org.opends.server.core.SearchOperation#processSearch
ds-cfg-enabled: true
ds-cfg-throwable-stack-frames: 100



We expect two things:

	Logging only from SearchOperation's processSearch method
	stack traces from every exception



In fact we get a firehose of logging from (it seems) every class in the server. So the notation to specify logging from a particular method seems to be wrong.

Changing the debug target to just have a scope of the class name seems to work correctly.

We also just get an exception message with no stack trace. e.g.



[12/Jun/2020:09:56:46 +0000] 2399038 trace thread={Worker Thread 1(107)} threadDetail={parentThread=main(1) isDaemon=false clientConnection=LDAP client connection from 1.2.3.4:49290 to 2.3.4.5:1636 operation=SearchOperation(connID=167584, opID=479024, baseDN=cn=changelog, scope=one, filter=(&(changeNumber>=2588532)(changeNumber<=2593531))) } method={processSearch(SearchOperation.java:898)}  caught={java.lang.NullPointerException}
