a reference can have the following optional searchFilter


Reference mapping objects have the following fields.
...
(Optional) "searchFilter": specifies the LDAP filter to use to search for the referenced entry. The default is "(objectClass=*)".

i've specified a filter like:


                    "members": {
                        "reference": {
                            "ldapAttribute": "uniqueMember",
                            "baseDN": "dc=mycompany,dc=loc",
                            "primaryKey": "uid",
                            "searchFilter": "(ou=live)",
                            "mapper": {
                                "object": {
                                    "_id": {
                                        "simple": {
                                            "ldapAttribute": "uid",
                                            "isSingleValued": true,
                                            "isRequired": true
                                        }
                                    },
                                    "displayName": {
                                        "simple": {
                                            "ldapAttribute": "cn",
                                            "isSingleValued": true,
                                            "writability": "readOnlyDiscardWrites"
                                        }
                                    }
                                }
                            }
                        }
                    },



the searchFilter is completely ignored.
