After upgrading a server (<= 3.0.0) with an HTTP Connection Handler enabled to new version (>= 3.5.0) the following components are not removed or missing after the upgrade:

1. the "config/http-config.json" file is not removed
2. the new json configuration files are not copied => "opendj/config/rest2ldap" folder does not exists
3. no default endpoint configured:

$ ./bin/dsconfig -p 4444 -D "cn=directory manager" -w p -h localhost -X list-http-endpoints -n
HTTP Endpoint : Type : enabled
--------------:------:--------


4. no default http authorization mechanism configured:

$ ./bin/dsconfig -p 4444 -D "cn=directory manager" -w p -h localhost -X list-http-authorization-mechanisms -n
HTTP Authorization Mechanism : Type
-----------------------------:-----
