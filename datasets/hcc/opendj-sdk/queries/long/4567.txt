dsconfig create-component with the --offline argument won't print the success message "The <component name>  was created successfully". 

Reproduce case: 

dsconfig delete-log-publisher --publisher-name "Json File-Based HTTP Access Logger" \
    -h <your local host> -p 4444 -w password -D "cn=Directory Manager" -X --no-prompt

dsconfig create-log-publisher --publisher-name "Json File-Based HTTP Access Logger" --type "json-file-http-access" --set enabled:true \
    -h <your local host> -p 4444 -w password -D "cn=Directory Manager" -X --no-prompt


 
 You'll see that no success message is printed.

But if you remove the --no-prompt argument, a success message will be printed.
