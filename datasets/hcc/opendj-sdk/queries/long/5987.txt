I noticed this while building training around profiles, but the online help for a specific profile is inaccurate.



$ ./setup --help-profile am-config
AM configuration data store (6.5.0) parameters
-----------------------
Parameter: backendName
Description: Name of the backend for storing config
Syntax: Name
Default: --set am-config:6.5.0/backendName:cfgStore
-----------------------
Parameter: baseDn
Description: The base DN to use to store AM's configuration in
Syntax: DN
Default: --set am-config:6.5.0/baseDn:ou=am-config
-----------------------
Parameter: amConfigAdminPassword
Description: Password of the administrative account that AM uses to bind to
OpenDJ
Syntax: Password



The default value for optional parameters cannot be used as such with the setup command.
And there is no help on how to use the mandatory parameters (which seems inconsistent).

Here's what happens if trying to use the exact string as displayed in `Default:`



./setup -w secret12 -h `uname -n` -p 1389 --adminConnectorPort 4444 --profile am-config --set am-config:6.5.0/baseDn:ou=MyAMConfig --set am-config:6.5.0/backendName:AMCfgStore --set am-config/amConfigAdminPassword:secret12
Error: Parameter(s) 'am-config' provided for profile 'am-config' is (are) not
used in the profile


And here's the command that works:



./setup -w secret12 -h `uname -n` -p 1389 --adminConnectorPort 4444 --profile am-config --set am-config/baseDn:ou=MyAMConfig --set am-config/backendName:AMCfgStore --set am-config/amConfigAdminPassword:secret12
