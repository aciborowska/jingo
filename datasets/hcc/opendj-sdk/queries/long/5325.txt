Bug description

I am seeing divergences  between RS's changelogs in a split DS/RS topology. The initial state is DS1-RS1-RS2-DS2 topology with version 5.5.0. After few operations (add, mod, moddn and del) on both DSs and after we upgrade RS1. After upgrade the changelog differs.
How to reproduce the issue

	Setup topology - 2DS 2RS - check attached script 
	Make few operations to both DSs - check attached script 
	Upgrade RS1 /path/to/rs1/opendj/upgrade -n --acceptLicense --force
	Check changelogs with following ldapsearch: 

/path/to/ds/opendj/bin/ldapsearch -h localhost -p 1398 -D "cn=Directory Manager" -w password -b "cn=changelog" "(objectClass=*)"




Command for test framework

./run-pybot.py -s replication_group3.upgrade -t Upgrade_Ds_Rs -v opendj  <- newer test using directly ./setup replication-server
./run-pybot.py -s replication_group3.upgrade -t Replication_Topology_Split_DS_RS -v opendj  <- older test which deletes backend to have kind of PURE RS
Test steps should be exactly same except the RS installation and topology creation. Result is same divergence in test log.

 or run the attached shell script - need to copy only zips in the same folder as the script.

No divergences across RS's changelogs at any point even if one is version 5.5.0 and other is 6.5.0.
