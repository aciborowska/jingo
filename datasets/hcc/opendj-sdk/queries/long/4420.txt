When setting disk-low-threshold or disk-full-threshold settings for a backend to a value that is higher than current free disk space, users can still add entries to the backend.

Steps to reproduce:

	Setup DJ (4.0.0), have a non-root user who can add entries


bin/ldapmodify -h localhost -p 1389 -D "cn=Directory Manager" -w password	
dn: ou=people,dc=example,dc=com 
changetype: modify 
add: aci 
aci: (target ="ldap:///ou=people,dc=example,dc=com") (targetattr != "userPassword")(version 3.0;acl "User uid=user.0,ou=people,dc=example,dc=com access"; allow(all)(userdn = "ldap:///uid=user.0,ou=people,dc=example,dc=com");)






	Set disk-low-threshold to a value that is higher than current free disk space


bin/dsconfig -h localhost -p 4444 -D "cn=Directory Manager" -w password -X set-backend-prop --backend-name "userRoot" --set disk-low-threshold:"63 gb" --advanced -n





	Add an entry to the DB


bin/ldapmodify -h localhost -p 1389 -D "uid=user.0,ou=people,dc=example,dc=com" -w password	
dn: uid=user.100,ou=people,dc=example,dc=com 
changetype: add 
objectclass: inetorgperson 
cn: User Additional 
sn: Additional 
uid: user.100




Expected result:

53 (Unwilling to perform)

Current result:

The entry gets added to the DB
