When configuring replication, if the 'adminPassword' attribute is provided with a password from the server's dictionary, the error message wrongly states that the password value for the attribute "userPassword" was unacceptable.

Ideally it should point out the attribute "adminPassword" rather than "userPassword"

Please see below:


[support@localhost bin]$ ./dsreplication  configure  --adminUID admin  --adminPassword password  --baseDN dc=example,dc=com  --host1 172.24.10.232  --port1 4444  --bindDN1 "cn=Directory Manager"  --bindPassword1 password  --replicationPort1 8989  --host2 172.24.10.233  --port2 4444  --bindDN2 "cn=Directory Manager"  --bindPassword2 password  --replicationPort2 8989  --trustAll  --no-prompt


Establishing connections ..... Done
Checking registration information ..... 
Error updating registration information.  Details: Registration information
error. Error type: 'ERROR_UNEXPECTED'. Details: Constraint Violation: The
password value for attribute userPassword was found to be unacceptable: The
provided password contained a word from the server's dictionary
See /tmp/opendj-replication-1485029234395058349.log for a detailed log of this
operation. 

 

 

Please note that DS is installed in Production mode.
