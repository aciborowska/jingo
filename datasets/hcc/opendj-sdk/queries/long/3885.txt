When the request router doesn't find a backend able to serve a request for a specific DN, it is routing the request to a fake NO_BACKEND in the hope this one doesn't exists.

If this one actually exists, then all request which are not matching any naming context will be forwarded to it. Results of these request routed to this backend are undetermined (aka: not tested).

While first seen as very unlikely to happen Chris Ridd actually found an interesting naming scheme where backend are named based on their country data: FR_BACKEND, GB_BACKEND, er, NO_BACKEND, ...
