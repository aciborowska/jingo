Under cn=monitor, proxy backend entries use an integer syntax for the attribute ds-mon-backend-is-private whereas the schema in 02-config.ldif says it should use a boolean syntax.
This issue is specific to proxy backend monitor entries, the entries for other backend types use the boolean syntax.

Steps to reproduce: 

	Setup a proxy server
	Search the proxy backend monitor entry

$ ./opendj/bin/ldapsearch -h localhost -p 1389 -D "uid=admin" -w password -b "cn=backends,cn=monitor" "objectClass=ds-monitor-backend-proxy"
dn: ds-cfg-backend-id=proxyRoot,cn=backends,cn=monitor
objectClass: top
objectClass: ds-monitor
objectClass: ds-monitor-backend
objectClass: ds-monitor-backend-proxy
ds-cfg-backend-id: proxyRoot
ds-mon-backend-is-private: 0
ds-mon-backend-proxy-base-dn: ou=people,dc=example,dc=com
ds-mon-backend-proxy-base-dn: ou=whatever,dc=example,dc=com
ds-mon-backend-writability-mode: enabled
