## Bug Report

### Steps to Reproduce:
run this profile

```
{
   "schemaVersion": "0.1",
   "fields": [
       {
           "name": "shortString"
       }
   ],
   "rules": [
       {
           "constraints": [
               {
                   "field": "shortString",
                   "is": "ofType",
                   "value": "string"
               },
               {
                   "field": "shortString",
                   "is": "shorterThan",
                   "value": 2
               },
               {
                   "not" : {
                       "field": "shortString",
                       "is": "null"
                   }
               },
               {
                   "field": "shortString",
                   "is": "matchingRegex",
                   "value": "[a-b|y-z]{5,10}"
               }
           ]
       }
   ]
}
```

### Expected Result:
Message saying the profile is contradictory

### Actual Result:
Generation runs forever and never generates

### Additional notes
The regex library we have has the option to check if nothing can be produced 
```automaton.isEmpty()```
but we are not making that check in the right place (during the FieldSpec merge).
