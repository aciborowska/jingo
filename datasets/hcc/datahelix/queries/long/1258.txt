## Bug Report

### Steps to Reproduce:
```
{
  "schemaVersion": "0.1",
  "fields": [
    {
      "name": "cusip"
    }
  ],
  "rules": [
    {
      "constraints": [
        {
          "field": "cusip",
          "is": "ofType",
          "value": "CUSIP"
        },
        {
          "not": {
            "field": "cusip",
            "is": "null"
          }
        }
      ]
    }
  ]
}
```

### Expected Result:
Valid CUSIPs

### Actual Result:
Invalid CUSIPs (according to https://www.getnewidentity.com/validate-cusip.php )

### Note:
The first six digits of the CUSIP identify the "issuer". It seems that if the issuer given is invalid, then the whole checksum is valid. The official list of issuers doesn't seem to be easily available, so (if confirmed) we may have to deprecate CUSIPs, or force the user to input valid issuers.
