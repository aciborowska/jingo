## Bug Report

### Steps to Reproduce:
Run the following profile with the command `java -jar generator.jar generate --max-rows=100 --replace profile.json output.csv`

[profile-1.txt](https://github.com/ScottLogic/datahelix/files/2987438/profile-1.txt)

### Expected Result:
Column 1 containing numbers between 1000 and 2000 and between 6000 and 7000. Column 2 being populated with "A type" for rows when column 1 has a value between 1000 and 2000.

### Actual Result:
Column 1 only contains numbers between 6000 and 7000. Column 2 is never populated.

### Environment:
Release V0.0.5
