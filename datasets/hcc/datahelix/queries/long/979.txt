## Bug Report

### Steps to Reproduce:
Run the following profile with the `--violate` flag. The profile already exists in the project, so the results can be seen using:
 `java -jar generator/build/libs/generator.jar generate --violate=true --replace examples\hard-contradiction-type-validation-2\profile.json outputs`. 
```
{
  "schemaVersion": "0.1",
  "fields": [
    {
      "name": "Column 1"
    }
  ],
  "rules": [
    {
      "rule": "Column 1 is a string",
      "constraints": [
        {
          "allOf": [
            {
              "field": "Column 1",
              "is": "ofType",
              "value": "string"
            },
            {
              "field": "Column 1",
              "is": "ofType",
              "value": "integer"
            },
            {
              "not": {
                "field": "Column 1",
                "is": "null"
              }
            }
          ]
        }

      ]
    }
  ]
}

```

### Expected Result:
This is a hard contradiction, generating no data. Therefore, the violation of this should be the universal set, including date time values. 

### Actual Result:
A single csv file is generated with only string and integer fields, and no date times.
