## Bug Report
A before constraint was set using the valid date 0000-01-01T00:00:00.000, however the error returned indicates that this date is not in the correct ISO format.

> Scenario: Running a 'before' request that specifies the lowest valid system date should only generate null data
>      Given foo is before 0000-01-01T00:00:00.000
>      Then the following data should be generated:
>        | foo                     |
>        | null                    |

### Steps to Reproduce:
 1. Run the above scenario in IntelliJ (or run the before.feature file from branch 309-before-CI)
 2. See that the scenario fails
 3. Review the error message of the scenario

### Expected Result:
The app should recognise the date of 0000-01-01T00:00:00.000 as valid and generate data accordingly (in this scenario instance then null data should be created).

### Actual Result:
The date is not recognised and a formatting error is thrown:

> java.lang.AssertionError: Exceptions thrown during generation
> Expected: an empty collection

> 	at org.hamcrest.MatcherAssert.assertThat(MatcherAssert.java:20)
> 	at org.junit.Assert.assertThat(Assert.java:956)
> 	at com.scottlogic.deg.generator.cucumber.steps.GeneralTestStep.theFollowingDataShouldBeGenerated(GeneralTestStep.java:156)

### Environment:
Scenario ran through IntelliJ run on Windows 10, run in full generation mode.
