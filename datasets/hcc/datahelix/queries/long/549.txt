## Bug Report

### Steps to Reproduce:
Using profile:
```json
{
  "schemaVersion": "v3",
  "description": "ShorterThan Test",
  "fields": [
    {
      "name": "column1"
    }
  ],
  "rules": [
    {
      "rule": "column1 is a string",
      "constraints": [
        {
          "field": "column1",
          "is": "ofType",
          "value": "string"
        },
        {
          "anyOf": [
            {
              "field": "column1",
              "is": "shorterThan",
              "value": 2
            },
            {
              "field": "column1",
              "is": "shorterThan",
              "value": 3
            }
          ]
        },
        {
          "field": "column1",
          "is": "containingRegex",
          "value": "[%]{1}"
        }
      ]
    }
  ]
}

```

execute generator with command:
`
java -jar ..\release-0.0.0\generator.jar generate shorterthan-bug.json.txt shorterthan-bug.csv -t FULL_SEQUENTIAL -w REDUCTIVE
`

then execute generator with command:
`java -jar ..\release-0.0.0\generator.jar generate shorterthan-bug.json.txt shorterthan-bug.csv -t FULL_SEQUENTIAL -w CARTESIAN_PRODUCT
`

### Expected Result:
results from each execution should match

### Actual Result:
when executed with `REDUCTIVE` walker, generator crashes with exception
**`
      Can't generate all strings for a non-finite regex
`**

when executed with `CARTESIAN_PRODUCT` results are output to csv file

### Environment:
generator.jar file produced with maven on Windows 10 command line (`cmd`)
