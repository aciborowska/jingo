## Bug Report

### Steps to Reproduce:
Run the generator against the profiles in `schemas\src\test\resources\test-profiles`.
Each valid profile should work, but some do not and produce validation errors.

### Expected Result:
Each valid profile should work

### Actual Result:
Validation errors are produced for valid profiles

### Environment:
WSL under windows 10 running the following script:
```
#!/bin/bash
BASE_DIR=/home/andy/src/datahelix/schemas/src/test/resources/test-profiles
VALIDATOR=leadpony

mkdir validator-$VALIDATOR
for f in $BASE_DIR/*.json
do
	echo "testing file : $f"
    java -jar /home/andy/src/datahelix-0.0.0/generator.jar generate $f $f-$VALIDATOR.out > $f-$VALIDATOR.log 2>&1
    mv $f-$VALIDATOR.* validator-$VALIDATOR
done

```
