There is a logical error in the following function in DateSerializer.java when source read '-1'
function is:

public Date deserialize(Date reuse, DataInputView source) throws IOException {
	long v = source.readLong();
	if(v == -1L) {
		return null;
	}
	reuse.setTime(v);
	return reuse;
}


when call this function for first time, if return null, then 'reuse' will be set null by caller;
when call this function for second time,if 'v!=-1' ,reuse.setTime(v) will throw NPE.
