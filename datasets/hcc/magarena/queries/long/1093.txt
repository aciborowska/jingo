If the Firemind Top Decks category is selected first so that the latest decks are downloaded this should fix the issue UNLESS a top deck recorded in the stats database is not included in the latest firemind decks download. 

Not sure why I decided not to include the firemind decks folder in the import but also more generally the code for getting the top decks needs to be more robust such that if a deck file referenced in the stats database cannot be found it should just skip over it and move on to the next deck instead of raising an exception.
