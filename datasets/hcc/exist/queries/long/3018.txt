### What is the problem

When using eXist's XQuery Update Facility, the `update delete expr` action leaves behind unexpected whitespace from the deleted node in the XML document. 

### What did you expect

When deleting a node, interior whitespace from that node should not be left behind as an artifact; the entire node should be deleted.

### Describe how to reproduce or add a test

From my question at: https://stackoverflow.com/questions/57937743/xquery-delete-leaves-empty-lines-in-xml-document-how-to-remove-them-exist-db/57944539#57944539:

I have an operation that deletes nodes from a stored XML document that looks like this:

```xml
<listBibl xmlns="http://www.tei-c.org/ns/1.0" xml:id="bibliography">
    <biblStruct type="book" xml:id="Z-BF2WLW8Y">
        <monogr>
            <title level="m">footitle1</title>
            <author>
                <name>author name</name>
            </author>
            <imprint>
                <publisher>some city</publisher>
                <date>2019</date>
            </imprint>
        </monogr>
    </biblStruct>
    <biblStruct type="book" xml:id="Z-4KF7YNP3">
        <monogr>
            <title level="m">footitle2</title>
            <author>
                <name>author name</name>
            </author>
            <imprint>
                <publisher>some other city</publisher>
                <date>2018</date>
            </imprint>
        </monogr>
    </biblStruct>
</listBibl>
```

The following function:

```xquery
declare local:delete-bibl()
{
   let $bibdoc := doc("/db/apps/myapp/data/list_bibliography.xml")
   for $bib in $bibdoc//tei:biblStruct[@xml:id = "Z-BF2WLW8Y"]
   return update delete $bib
};
```

leaves the XML file with whitespace like this:

```xml
<listBibl xmlns="http://www.tei-c.org/ns/1.0" xml:id="bibliography">
    
    
    
    
    
    
    
    
    
    
    
    
    <biblStruct type="book" xml:id="Z-4KF7YNP3">
        <monogr>
            <title level="m">footitle2</title>
            <author>
                <name>author name</name>
            </author>
            <imprint>
                <publisher>some other city</publisher>
                <date>2018</date>
            </imprint>
        </monogr>
    </biblStruct>
</listBibl>
```

The 10 newlines in the deleted `<biblStruct>` element should have been deleted by the `update delete` action, but they were instead retained.

### Context information

- eXist-db version: 4.7  (installed with .dmg)
- Java 8 update 101
- Operating system: macOS 10.11.6
- No customized configurations
