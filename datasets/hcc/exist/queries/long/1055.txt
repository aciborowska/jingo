We are using the diacritics="no" filter on our Lucene index.  When searching for a name with diacritics and a wildcard I get no results, while searching for the same name without diacritics and a wildcard will return the expected results. 

Here are a few links demonstrating the various results:

Full name without diacritics and a wildcard: 
http://syriaca.org/geo/search.html?q=abajal* 

Diacritics and wildcard:
http://syriaca.org/geo/search.html?q=%C4%80b%C4%81j%C4%81l*

Here is our index configuration file:
https://github.com/srophe/srophe-app-data/blob/master/collection.xconf

Here is a sample record: 
https://github.com/srophe/srophe-app-data/blob/master/data/places/tei/1949.xml

And a simplified version of the query:

```
xquery version "3.0";
declare namespace tei = "http://www.tei-c.org/ns/1.0";

for $r in collection('/db/apps/srophe-data/data/places/tei')//tei:placeName[ft:query(.,"Ābājāl*")]
return $r
```
