### What is the problem

When using the ant task to restore a directory I get the following error:

```
BUILD FAILED
/home/mike/exist_ant_tasks/build.xml:28: xdb:restore doesn't support the "dir" attribute
```

### What did you expect
I'd expect the task to work and the data to be restored.

### Describe how to reproduce or add a test

According to the documentation "dir" is an acceptable attribute for the restore task:
http://www.exist-db.org/exist/apps/doc/ant-tasks.xml#D2.2.14

I'm using the following build file:

```
<?xml version="1.0" encoding="UTF-8"?>
<project default="Backup" xmlns:xdb="http://exist-db.org/ant">

    <!-- the properties file -->
    <property file="build.properties"/>

    <!-- classpath with eXist jars -->
    <path id="classpath.core">
        <fileset dir="${server.dir}/lib/core">
            <include name="*.jar"/>
        </fileset>
        <pathelement path="${server.dir}/exist.jar"/>
        <pathelement path="${server.dir}/exist-optional.jar"/>
    </path>

    <!-- include exist ant tasks -->
    <typedef resource="org/exist/ant/antlib.xml" uri="http://exist-db.org/ant">
        <classpath refid="classpath.core"/>
    </typedef>

    <!-- backup (collection) -->
    <target name="Backup" description="Backup collection to a directory">
        <xdb:backup uri="${backup.uri}" dir="${backup.dir}" user="${backup.user}" password="${backup.pass}"/>
    </target>

    <!-- restore (collection) -->
    <target name="Restore" description="Restore collection to a directory">
        <xdb:restore uri="${backup.uri}" user="${backup.user}" password="${backup.pass}" dir="${backup.dir}"/>
    </target>
</project>
```

Can Ant correctly convert an attribute value to a java.nio.file.Path, as used in the public setter method for the custom task?

https://github.com/eXist-db/exist/blob/develop/src/org/exist/ant/RestoreTask.java
