### What is the problem

eXist should reject `fn:concat#1` with error XPST0017 (https://www.w3.org/TR/xpath-30/#ERRXPST0017), since the concat function requires an arity of 2 or more. Instead, it serializes `fn:concat#1` as `fn:concat#-1`. (`fn:concat#2` correctly returns no error.)

### What did you expect

I expected eXist to reject `fn:concat#1` with error XPST0017 (https://www.w3.org/TR/xpath-30/#ERRXPST0017). 

### Describe how to reproduce or add a test

```xquery
xquery version "3.1";

declare namespace output="http://www.w3.org/2010/xslt-xquery-serialization";
declare option output:method "adaptive";

fn:concat#1
```

eXist returns the following result without raising an error:

> fn:concat#-1

BaseX returns the following error:

> [XPST0017] fn:concat(atom1,atom2[,...]): 1 argument supplied.
