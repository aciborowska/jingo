**Describe the bug**

`util:get-resource-by-absolute-id` does return an empty sequence when given a valid absolute resource id.

**Expected behavior**

```xquery
util:get-resource-by-absolute-id(
    util:absolute-resource-id('/db/some/file'))
```

to return '/db/some/file' when this file exists in the database.

**To Reproduce**

```Xquery
xquery version "3.1";

module namespace abs="http://exist-db.org/xquery/absolute-resource-id";

declare namespace test="http://exist-db.org/xquery/xqsuite";

declare variable $abs:XML := document { <root/> };

declare
    %test:setUp
function abs:setup() {
    let $testCol := xmldb:create-collection("/db", "test")
    return xmldb:store("/db/test", "test.xml", $abs:XML)
};

declare
    %test:tearDown
function abs:tearDown() {
    xmldb:remove("/db/test")
};

declare
    %test:assertEquals('/db/test/test.xml')
function abs:test() {
    util:get-resource-by-absolute-id(
        util:absolute-resource-id('/db/test/test.xml'))
};
```

**Context (please always complete the following information):**
 - OS: OSX 10.15.4
 - eXist-db version: 5.3.0-SNAPSHOT 093c4c0726e02868acadbce9583ec5809f462b33
 - Java Version 1.8.0_152-b16

**Additional context**
- How is eXist-db installed? built from source 
- Any custom changes in e.g. `conf.xml`? none
