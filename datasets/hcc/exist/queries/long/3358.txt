**Describe the bug**

Passing a function with a wrong arity (not one) as the second parameter to a higher order function such as `fn:for-each` or `fn:filter` will throw a java exception.

```
ERROR (XQueryServlet.java [process]:559) - null
java.lang.ArrayIndexOutOfBoundsException: null
```

`fold-left` and `fold-right` not tested yet but are likely also affected, judging from the implementation.


**Expected behavior**

XQuery exception XPTY0004 - (argument) type mismatch

Saxon 10 HE returns `XPTY0004  The supplied function (function($m, $n){ $m }) has 2 arguments - expected 1`

**To Reproduce**

```Xquery
xquery version "3.1";

module namespace t="http://exist-db.org/xquery/test";

declare namespace test="http://exist-db.org/xquery/xqsuite";

declare
    %test:assertError("XPTY0004")
function t:for-each-arity-0() {
    for-each(1, function () { 1 })
};

declare
    %test:assertError("XPTY0004")
function t:for-each-arity-2() {
    for-each(1, function ($a, $b) { 1 })
};

declare
    %test:assertError("XPTY0004")
function t:filter-arity-0() {
    filter(1, function () { 1 })
};

declare
    %test:assertError("XPTY0004")
function t:filter-arity-2() {
    filter(1, function ($a, $b) { 1 })
};

```

 - Java Version 1.8.0_152 64-Bit
