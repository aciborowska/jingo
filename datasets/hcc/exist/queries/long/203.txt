The following template:

```
<tr class="templates:each?from=addresses&amp;to=address">
    <td class="ex:print-name"/>
    <td class="ex:print-street"/>
    <td class="ex:print-city"/>
</tr>
```

results in the following markup:

```
<tr class="templates:each?from=addresses&amp;to=address">
                                                </tr><tr class="templates:each?from=addresses&amp;to=address">
                    <td class="ex:print-name">111</td>
                    <td class="ex:print-street">222</td>
                    <td class="ex:print-city">333</td>
                </tr>
...more tr tags here...
```

The first empty tr tag is always added. This looks like a bug.

This example is taken from the documentation: 
http://exist-db.org/exist/apps/demo/examples/templating/templates.html
