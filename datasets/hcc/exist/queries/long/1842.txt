I used to check that external variables in scripts have been given a value by catching error `err:XPDY0002`.

Since 1713038fb1dece5338bc7fed5400c8f184378b42 it is not possible anymore.

With this script (`/db/apps/test.xql`):

```xquery
xquery version "3.1";

declare variable $local:var external;

if (try { $local:var != '' } catch err:XPDY0002 { false() }) then
    $local:var
else
    'undefined'
```

it works (once after the script has been saved or the server started) but fails on the next run (from the presumably now compiled script):

```
$ curl http://localhost:8080/exist/apps/test.xql
undefined
$ curl http://localhost:8080/exist/apps/test.xql
<?xml version="1.0" ?><exception><path>/db/apps/test.xql</path><message>err:XPDY0002 no value specified for external variable local:x</message></exception>
```

Tested with current eXist develop (3996133151ad5a0948a2f6fe0966e5791671d49a) on Linux with Java 1.8.0.
