### What is the problem

When (1) an arrow operator is used in function A and (2) the function that the arrow operator calls, function B, is declared after function A in the query prolog, eXist returns a "function not found" error.

### What did you expect

I expected eXist to locate function B when calling function B via the arrow operator. 

My tests below show that Saxon and BaseX do not have this problem.

My tests also show that eXist has no problem locating function B when we either (1) call function B without the arrow operator or (2) move function B above function A in the prolog.

### Describe how to reproduce or add a test

Given the following query, eXist raises a "Function not found: local:B" error:

```xquery
xquery version "3.1";

declare function local:A($x) { 
    let $y := $x || $x 
    return 
        $y => local:B()
};

declare function local:B($x) {
    string-length($x)
};

local:A("hello")
```

The full error:

> Cannot compile xquery: err:XPST0017 Function not found: local:B [at line 6, column 12, source: xquery version "3.1"; declare function local:A($x) { let $y := $x || $x return $y => local:B() }; declare function local:B($x) { string-length($x) }; local:A("hello")]

The query runs without error, as expected, in Saxon and BaseX.

The following variation, which defines `local:B` first, completes without error in eXist:

```xquery
xquery version "3.1";

declare function local:B($x) {
    string-length($x)
};

declare function local:A($x) { 
    let $y := $x || $x 
    return 
        $y => local:B()
};

local:A("hello")
```

Similarly, calling local:B without the arrow operator completes without error in eXist:

```xquery
xquery version "3.1";

declare function local:A($x) { 
    let $y := $x || $x 
    return 
        local:B($y)
};

declare function local:B($x) {
    string-length($x)
};

local:A("hello")
```
