**Describe the bug**

XQuery allows a function body to be empty. This is equivalent of a function body returning an empty sequence.

Evaluating the following line

```xquery
function () {}
```
yields a compilation error: `Syntax error within user defined function: unexpected token: }` (also: location information is missing)

```xquery
(function () {})()
```

Parses but throws when executed with `Cannot compile xquery: err:XPST0003 Syntax error within user defined function: unexpected token: } [at line 1, column 15]`


**Expected behavior**

The above to evaluate to a function with arity 0 returning an empty sequence.

Saxon10-HE handles this correctly:

```
âžœ java -cp saxon-he-10.0.jar net.sf.saxon.Query -qs:"(function () {})()"
<?xml version="1.0" encoding="UTF-8"?>%
```


**To Reproduce**

```Xquery
xquery version "3.1";

module namespace t="http://exist-db.org/xquery/test";

declare namespace test="http://exist-db.org/xquery/xqsuite";

declare
    %test:assertTrue
function t:test() {
  let $f := function () { () }
  return $f instance of function(*)
};

(: this will not compile :)
declare
    %test:assertTrue
function t:test() {
  let $f := function () {}
  return $f instance of function(*)
};

(: this will not run :)
declare
    %test:assertFalse
function t:test() {
  exists((function () {})())
};
```
