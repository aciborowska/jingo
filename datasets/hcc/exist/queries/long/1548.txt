### What is the problem

In the current develop branch, the use of an arrow operator inside user-defined functions is causing an NPE. Adapting the query to not use the arrow operator eliminates the NPE.

### What did you expect

I expected the arrow operator not to cause an NPE in this circumstance. Saxon and BaseX return the expected result, 6.

### Describe how to reproduce or add a test

The following query triggers the NPE:

```xquery
xquery version "3.1";

declare function local:foo($x) {
    local:foo($x, "bar")
};

declare function local:foo($x, $y) {
    ($x || $y) => string-length()
};

local:foo("foo")
```

The following query is identical to the first, but it avoids the use of the arrow operator. It does not trigger an NPE:

```xquery
xquery version "3.1";

declare function local:foo($x) {
    local:foo($x, "bar")
};

declare function local:foo($x, $y) {
    string-length($x || $y)
};

local:foo("foo")
```
