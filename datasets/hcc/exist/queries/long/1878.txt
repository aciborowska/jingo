### What is the problem

In the current release and the develop branch, eXist returns the wrong day of the week when using the `[F]` picture string to output the day of the week with `format-date()` or `format-dateTime()`. It consistently returns the day of the week *one day after* what it should be returning. So, instead of saying that today is Friday, it says Saturday.

### What did you expect

I expected eXist to return the correct day of the week.

### Describe how to reproduce or add a test

The problem is evident in my tests with the eXist 4.1.0 release (919514c68) and later (including the current develop HEAD, f53dbceae), but not in the eXist 4.0.0 release (cc66ebc). When I copy https://github.com/eXist-db/exist/blob/develop/test/src/xquery/dates/format-dates.xql into my database, I get the failures shown below. A quick test anyone can run is this:

```xquery
"Is today really " || format-date(current-date(), "[FNn]") || "?"
```
