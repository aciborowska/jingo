### What is the problem

When filtering a map entry with a postfix lookup, an error is thrown if the postfix lookup expression does not come first in the predicate.

### What did you expect

I expected a postfix expression to be valid regardless of its position in a predicate.

### Describe how to reproduce or add a test

The following query triggers the error:

```xquery
xquery version "3.1";

let $map := map { "name": "mike" }
return
    $map["mike" eq ?name]
```

BaseX and Saxon both return the expected result without error:

```xquery
map { "name": "mike" }
```

eXist returns the following error:

`Cannot compile xquery: exerr:ERROR <AST>:0:0: unexpected end of subtree`

eXist is able to complete the query if the postfix lookup is performed first in the predicate:

```xquery
xquery version "3.1";

let $map := map { "name": "mike" }
return
    $map[?name eq "mike"]
```
