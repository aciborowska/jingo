In 2.2 RC2 

setting the %output:media-type to application/json removes the default UTF-8 encoding from the HTTP Content-Type Header

I've found no other way to set the encoding - only omitting the media-type works

``` xquery
declare
  %rest:GET
  %rest:path("/categories/{$id}")
  (: removes the encoding from response :)
   %output:media-type("application/json") 
   %output:method("json")
  (: not needed: %rest:produces("application/json") :)
  (: gives error:  %output:media-type("application/json; charset=UTF-8") :)
  (: gives error:  %output:encoding("UTF-8") :)
function _:get-json($id as xs:string) {
     <json>umlauts: äöü</json>
};
```
