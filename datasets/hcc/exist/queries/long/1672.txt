`xmldb:document()`, `xmldb:copy()` produce unexpected errors:

1. unable to open an existing file
2. not finding existing collections
3. wrong user

Consider the following example, run from eXide on eXist 3.6.0, the remote running eXist 3.4.1

```xquery
    xquery version "3.1";
    
    let $auth2 := xmldb:login('xmldb:exist:///db/apps', 'admin', '')
    
    let $coll := xmldb:get-child-collections('xmldb:exist://dev2.hab.de:80/xmlrpc/db/edoc/')
    
    let $date := xmldb:last-modified('xmldb:exist://dev2.hab.de:80/xmlrpc/db/edoc', 'test.xml')
    let $size := xmldb:size('xmldb:exist://dev2.hab.de:80/xmlrpc/db/edoc/', 'test.xml')
    let $doc := xmldb:document('xmldb:exist://dev2.hab.de:80/xmlrpc/db/edoc/test.xml')
    let $doc2 := doc('http://dev2.hab.de/edoc/test.xml')
    
    let $coll2 := xmldb:get-child-collections('xmldb:exist:///db/apps/edoc')
    let $copy := 
        try {
            xmldb:copy('xmldb:exist://dev2.hab.de:80/xmlrpc/db/edoc', 'xmldb:exist:///db/apps/edoc', 'test.xml')
        } catch * {
            $err:code || ': ' || $err:description
        }
        
    let $user := xmldb:get-current-user()
    let $copy2 := 
        try {
            xmldb:copy('xmldb:exist://dev2.hab.de:80/xmlrpc/db/edoc', 'xmldb:exist:///db/apps', 'test.xml')
        } catch * {
            $err:code || ': ' || $err:description
        }
    
    return 
        <result>
            <collectionCount>{count($coll)}</collectionCount>
            <collection>{$coll}</collection>
            <date>{$date}</date>
            <size>{$size}</size>
            <document>{$doc}</document>
            <document2>{$doc2}</document2>
            <coll2>{$coll2}</coll2>
            <copy>{$copy}</copy>
            <user>{$user}</user>
            <copy2>{$copy2}</copy2>
        </result>
```

The remote machine's log for the time the query was run has 2 warnings by the transaction manager that 2 transactions were auto aborted but no further errors.

### What did you expect

- `$doc` should contain the file as does `$doc2`
- The first copy operation should be successful; as can be seen from `$coll2`, the collection exists and is accessible
- The second copy operation should be successful; as can be seen from `$user`, the current user is 'admin'
