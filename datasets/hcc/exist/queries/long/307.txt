It seems like the [QuartzSchedulerImpl initialization section](https://github.com/eXist-db/exist/blob/develop/src/org/exist/scheduler/impl/QuartzSchedulerImpl.java#L98) is wrong because the thread pool class name is invalid:

``` java
defaultQuartzProperties.setProperty("org.quartz.threadPool.class", "org.quartz.threadPool.class");
```

This would lead to a runtime exception (i.e. `java.lang.ClassNotFoundException: org.quartz.threadPool.class`)

Shouldn't the configuration be changed to something like this?:

``` java
defaultQuartzProperties.setProperty("org.quartz.threadPool.class", "org.quartz.simpl.SimpleThreadPool");
```
