When working on a collection of rdf-documents with various indexes I have noticed that the monitoring app fails listing the entries for indexed attributes having a namespace prefix such as @rdf:about and @rdf:resource.

The error message indicates a problem with QName:
err:XPTY0004 checking function parameter 2 in call util:log("DEBUG", concat("INDEXES qname     :    ", $indexes:qname)): Invalid lexical form of either prefix or local name. [at line 211, column 70, source: /db/apps/monex/modules/indexes.xqm]

I guess the problem is in the code starting with line 38 in indexes.xm where the attribute name including  '@' is used as second argument for QName().

Line 38: 

```
declare variable $indexes:qname := 
    if ($indexes:node-name ne '') then 
        if (not(matches($indexes:node-name, 'xml:')) and contains($indexes:node-name, ':')) then
            QName(indexes:get-namespace-uri-from-node-name($indexes:node-name, $indexes:collection), $indexes:node-name) 
        else 
            xs:QName($indexes:node-name)
    else ();
```

replacing $indexes:node-name with code that removes '@' if it is present seems to solve the problem:

```
declare variable $indexes:qname := 
    if ($indexes:node-name ne '') then 
        if (not(matches($indexes:node-name, 'xml:')) and contains($indexes:node-name, ':')) then
            QName(indexes:get-namespace-uri-from-node-name($indexes:node-name, $indexes:collection), 
            if (starts-with($indexes:node-name, '@')) then substring-after($indexes:node-name,'@') else $indexes:node-name) 
        else 
            xs:QName($indexes:node-name)
    else ();
```
