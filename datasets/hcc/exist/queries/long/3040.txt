### What is the problem
base-uri($node) shows the URI where the resource containing $node was stored, rather than the location it has been moved to. 

### What did you expect
base-uri($node) shows the URI for the node's new location


### Describe how to reproduce or add a test
Within a collection `_shared-prefix_`, create a collection `staging` containing a collection `data` and within the latter a resource such as 

```
<index xmlns="http://example.org/namespace">
    <item>A</item>
</index>
```

Create a collection, sister to `staging` called `prod`.

Create an index scoping over both source and target, such as
```
<collection xmlns="http://exist-db.org/collection-config/1.0">
    <index xmlns:mynamespace="http://example.org/namespace">
        <range>
            <create qname="mynamespace:item" type="xs:string"/>
        </range>
    </index> 
</collection>
```

Check that the following returns a uri containing `staging`: 

```
xquery version "3.1";
declare namespace ns ="http://example.org/namespace";
base-uri(collection("_shared-prefix_")//ns:item)
```

Execute

`xmldb:move("_shared-prefix_/staging/data", "_shared-prefix_/prod")`

The XQuery above (with version spec) should not return a URI containing `_shared-prefix_/staging`.
