When using a xml document which has xmlns:xml="http://www.w3.org/XML/1998/namespace" declaration, exist returns org.exist.xquery.XPathException: exerr:ERROR err:XQST0070 can not redefine 'xmlns:xml'.

But according to the spec, it should raise no error because the prefix is using the correct namespace. It is correct to raise an error if the namespace were different to "http://www.w3.org/XML/1998/namespace" 

I think the error is caused here https://github.com/eXist-db/exist/blob/develop/src/org/exist/xquery/ElementConstructor.java#L109

You can find an example of the bug here:

```
xquery version "3.0";
let $row := <parent xmlns:xml="http://www.w3.org/XML/1998/namespace">
                <dummyElement>blabla</dummyElement>
            </parent>
return $row
```
