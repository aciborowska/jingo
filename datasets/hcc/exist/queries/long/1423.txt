### What is the problem

In some cases, a local variable bound in a FLWOR expression is not readable in the WHERE clause.

An example is this query:
```xquery
    for $doc in xmldb:xcollection( "/db/tmp" )
    let $dn := "/db/tmp/" || util:document-name($doc)
    where (doc-available($dn))
    return true()
```

### What did you expect

The query should return a sequence of true()-values for each document in `/db/tmp`. 
Instead, the following error is reported:
```
err:XPDY0002 variable '$dn' is not set.
```

Some equivalent expressions in the WHERE-clause  do not cause the error, e.g. `where (doc-available($dn) or false())` works as expected.

### Describe how to reproduce or add a test

Here is an XQSuite unit test demonstrating the behavior:

```xquery
xquery version "3.0";
module namespace t="http://exist-db.org/xquery/test/issue1423";
declare namespace test="http://exist-db.org/xquery/xqsuite";

declare %test:assertEmpty
function t:failing_with_XPDY0002() {    
    for $doc in (/*)[1]
    let $dn := "/some/nonexisting/path"
    where (doc-available($dn))
    return true()
};

(: the following tests are successful: :)

declare %test:assertEmpty
function t:thisWorks() {
    for $doc in (/*)[1]
    let $dn := "/some/nonexisting/path"
    where (doc-available($dn) or false())  (: only difference: ... or false() :)
    return true()
};

declare %test:assertEmpty
function t:thisAlsoWorks() {
    for $doc in <some-element/> (: iterate over in-memory elements, not db docs :)
    let $dn := "/some/nonexisting/path"
    where (doc-available($dn))
    return true()
};
```

I tried various functions, but only `doc-available` in the WHERE-clause seemed to trigger the error.
