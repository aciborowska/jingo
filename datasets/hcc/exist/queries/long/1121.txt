This simple query returns two items as expected:

``` xquery
declare function local:test() { (1,<n/>) };
local:test()
```

However, simply adding an variable declaration causes a mysterious XPTY0018 error, eg

``` xquery
declare variable $a := 'a';
declare function local:test() { (1,<n/>) };
local:test()
```

results in the error message 

> "err:XPTY0018 Cannot mix nodes and atomic values in the result of a path expression. [at line 1, column 24, source:...]"
