### What is the problem

The following query raises an error:

```xquery
("x", "y") => subsequence(1, 1) => matches("x")
```

The error:

```text
2019-12-06 14:08:51,839 [qtp1729081400-44] ERROR (XQueryServlet.java [process]:559) - org.exist.xquery.value.StringValue cannot be cast to org.exist.xquery.value.NodeValue 
java.lang.ClassCastException: org.exist.xquery.value.StringValue cannot be cast to org.exist.xquery.value.NodeValue
	at org.exist.xquery.value.SubSequence.isPersistentSet(SubSequence.java:377) ~[exist-core-5.1.0.jar:5.1.0]
	at org.exist.xquery.functions.fn.FunMatches.eval(FunMatches.java:322) ~[exist-core-5.1.0.jar:5.1.0]
...
```

### What did you expect

I expected the query to return `true()` instead of an error.  

The following slight variants of the query do return the expected result:

```xquery
("x", "y") => subsequence(1, 1) => string() => matches("x")
```

```xquery
("x", "y") => head() => matches("x")
```

The problem seems specific to `fn:subsequence()` and its handling of data types.
