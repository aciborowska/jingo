We have upgraded an application from Micronaut 1.2.0.RC1 to 1.2.0.RC2 and one test fails now.

We have controller actions annotated with `@Version`. The controller action uses a POJO to do the binding. If we add the `@Header` parameter as the second argument in the method everything works. On the other hand, if we define the `@Header` parameter as the first one in the controller action, Micronaut tries to bind that to the POJO and it fails.

The test for the first action pass and the second one fails.

```java
    @Version("1.0")
    @Post("/version-header-last")
    public Book versionHeaderLast(@Valid Book book, @Header(VERSION_HEADER) String apiVersion) {
        return new Book(book.getTitle().toUpperCase());
    }

    @Version("1.0")
    @Post("/version-header-first")
    public Book versionHeaderFirst(@Header(VERSION_HEADER) String apiVersion, @Valid Book book) {
        return new Book(book.getTitle().toUpperCase());
    }
```
