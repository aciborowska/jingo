Hi all,

Submit an empty value to a @Nullable Long fail with message ( "Failed to convert argument [quantity] for value [] due to: For input string: \"\"" )

@Post(uri = "/long", consumes = MediaType.MULTIPART_FORM_DATA)
public HttpResponse<?> longVersion(@Body @Nullable Long quantity) {
	return HttpResponse.ok(CollectionUtils.mapOf("Success Quantity", quantity));
}

Submit an empty value to a @Nullable String work well

@Post(uri = "/string", consumes = MediaType.MULTIPART_FORM_DATA)
public HttpResponse<?> stringVersion(@Body @Nullable String firstname) {
	return HttpResponse.ok(CollectionUtils.mapOf("Success Firstname",  firstname));
}

Steps
1. - clone https://github.com/ziguane/nullable.git to reproduce the issue
2. - cd nullable
3. - ./gradlew run
4. - Browse http://localhost:8082/
5. - Submit form 1 with empty String -> success
6. - Submit form 2 with String setted to 'Bob' -> success
7. - Submit form 3 with empty Long -> fail
8. - Submit form 4 with Long setted to 123 -> success

### Environment Information

- **Operating System**: MacOS Sierra 10.12.6 
- **Micronaut Version:** 1.2.0-RC1
- **JDK Version:** Java(TM) SE Runtime Environment (build 1.8.0_102-b14)

Bind value to String and convert it to Long is a workaround, but it is not really sexy.

How can we solve this problem ?
