### Steps to Reproduce

If you have the following:

```kotlin
@Client("/")
interface GreetingClient {

    @GetMapping("/greeting{?name}")
    fun greet(name : String? ) : Greeting
}
```

Compilation will fail with:

```
GreetingClient.java:11: error: The uri variable [name] is optional, but the corresponding method argument [java.lang.String name] is not defined as an Optional or annotated with the javax.annotation.Nullable annotation.
[ERROR]     public abstract greeting.example.Greeting greet(@org.jetbrains.annotations.Nullable()
```

Even though the JetBrains version of `Nullable` is there.

Workaround is to remove `micronaut-validation` from the annotation processor path.

### Expected Behaviour

Should compile

### Actual Behaviour

Doesn't compile

### Environment Information

- **Micronaut Version:** 1.0.2
