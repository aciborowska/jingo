Applications using Micronaut Data (both JPA or JDBC) and Micronaut 2.0 snapshot fail to start with a `BeanInstantiationException`

I'm not 100% sure because this was detected by GraalVM CI pipeline and when this happened there were issues in Gitlab infrastructure but I think the change that broke this is between these commits:

https://github.com/micronaut-projects/micronaut-core/compare/cfba3bd2...8da5e5be


### Steps to Reproduce

- `git clone https://github.com/micronaut-graal-tests/micronaut-data-jdbc-graal`
- `cd micronaut-data-jdbc-graal`
- `git checkout master_h2`
- `./gradlew run`


### Expected Behaviour

The app should start.
