We are deploying some Micronaut Microservices in a docker swarm. In a docker swarm the convention for naming a service is {STACK_NAME}_{SERVICE_NAME}. This means that services have an underscore in their name. When we use the `@Client` annotation to access a service, we get an "hostname can't be null" exception, because the given URL is parsed by URI class and URI class does not allow underscores in hostnames. But in this case, it's a valid URL. 

### Task List

#### Example:

```
@Singleton
public class ClientExample {

    private final RxHttpClient httpClient;

    public ClientExample(@Client("${api.url}") RxHttpClient httpClient) {
        this.httpClient = httpClient;
    }

    public void doExecute() {
        HttpRequest<?> httpRequest = HttpRequest.GET("/");
        httpClient.exchange(httpRequest).blockingFirst();
    }

}
```

And the application.yml:

```
micronaut:
  application:
    name: uriproblem

api:
  url: http://dockerstack_dockerservice
```

You can download an example application here:
https://github.com/danielplappert/micronaut-uriproblem

IllegalArgumentException: hostname can't be null
