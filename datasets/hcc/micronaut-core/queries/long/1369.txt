
### Steps to Reproduce

1. Setup a project using .properties files for configuration
2. Define a property that is a list with 3 values or more
3. start the application

### Expected Behaviour
the application should start (with the correct properties)

### Actual Behaviour

Micronaut crashes with an out-of-bounds exception


from debugging this, the issue seems to be in PropertySourcePropertyResolverSpec.java between lines 441 and 459. There seems to be a (false) assumption that an iterator from a Hashmaps keyset arrives in order. The order of such iterators is in fact not defined, and so we get an outofbounds exception.

One would have to sort the list beforehand to be able to do this. 

here's a stacktrace:
