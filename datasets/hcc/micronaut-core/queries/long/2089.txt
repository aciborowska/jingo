### Steps to Reproduce

1. Have `@CircuitBreaker` annotation on method that returns `Single.error(exception)`
2. Call the method multiple times, making sure to observe the exception in the `Single` using e.g. `.blockingGet()`.
3. `CircuitBreakerRetry.open()` **throws** the exception instead of returning a `Single.error(exception)` as expected.

### Expected Behaviour

`CircuitBreakerRetry.open()` should return `Single.error(lastError)` instead of `throw (RuntimeException) lastError;` when the expected return type is `Single`.

### Actual Behaviour

`CircuitBreakerRetry.open()` throws `(RuntimeException) lastError;`

### Environment Information

- **Operating System**: Linux
- **Micronaut Version:** 1.2.0
- **JDK Version:** OpenJDK 11
