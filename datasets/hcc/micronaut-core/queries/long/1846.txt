The heartbeat event is documented as enabled by default:

https://docs.micronaut.io/snapshot/guide/configurationreference.html#io.micronaut.health.HeartbeatConfiguration

and 

https://docs.micronaut.io/snapshot/api/io/micronaut/health/HeartbeatConfiguration.html

However, in Micronaut v 1.1.3 it is disabled by default and needs to be explicitly enabled in order to be used.

### Steps to Reproduce

1. Create a Micronaut 1.1.3 application. In our case we have a Kafka consuming app.
2. Implement a HeartbeatLogger which simply logs "logging heartbeat" when it receives a heartbeat event. Below is our HeartbeatLogger and a simplified version of our AppLogger:
```

import com.example.kafka.consumer.AppLogger;
import io.micronaut.context.event.ApplicationEventListener;
import io.micronaut.health.HeartbeatEvent;

import javax.inject.Inject;
import javax.inject.Singleton;
import java.util.Collections;

@Singleton
class HeartbeatLogger implements ApplicationEventListener<HeartbeatEvent> {

    @Inject
    private AppLogger appLogger;

    @Override
    public void onApplicationEvent(HeartbeatEvent event) {
        appLogger.info(Collections.EMPTY_MAP, "logging heartbeat");
    }

    @Override
    public boolean supports(HeartbeatEvent event) {
        return true;
    }
}
```

And the Logger:

```
import ch.qos.logback.classic.Logger;
import ch.qos.logback.classic.LoggerContext;
import org.slf4j.LoggerFactory;

import javax.inject.Singleton;
import java.util.Map;

import static net.logstash.logback.argument.StructuredArguments.entries;

@Singleton
public class AppLogger {

    public void info(Map context, String message) {
        getLogger("exampleKafkaConsumer").info(message, entries(context));
    }

    private Logger getLogger(String loggerName) {
        LoggerContext context = (LoggerContext) LoggerFactory.getILoggerFactory();
        return context.getLogger(loggerName);
    }
}
```
3. Run your application and wait for more than 15 seconds. The default micronaut.heartbeat.interval is 15 seconds, but we can choose to overwrite it to 1 second to reduce our waiting time. 

4. Check your log file. 

### Expected Behaviour

You should see the "logging heartbeat" message in your log file.

### Actual Behaviour

The "logging heartbeat" message is not logged.

### Environment Information

- **Operating System**: macOS Sierra v 10.12.6
- **Micronaut Version:** 1.1.3
- **JDK Version:** 1.8.0_152
