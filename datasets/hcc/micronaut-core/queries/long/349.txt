### Steps to Reproduce

Given a client like:

```groovy
    @Client("/itunes")
    static interface ItunesClient {

        @Get("/search?limit=25&media=music&entity=album&term={term}")
        SearchResult search(String term)
    }
```

The request sent will ignore the query parameters.

### Expected Behaviour

The query parameters should be included

### Actual Behaviour

The query parameters are not included

### Environment Information

- **Operating System**: ALL
- **Micronaut Version:** 1.0.0.M2
- **JDK Version:** ALL

### Cause

The `resolveRequestURI` method is ignoring the original query parameters when a context path is set:

https://github.com/micronaut-projects/micronaut-core/blob/master/http-client/src/main/java/io/micronaut/http/client/DefaultHttpClient.java#L908
