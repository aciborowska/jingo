### Steps to Reproduce

Having a repository class like:

```
@Singleton
@Transactional(readOnly = true)
public class ContinentRepository extends AbstractQuerydslRepository<Continent> {

    @Inject
    public ContinentRepository(final SessionFactory sessionFactory) {
        super(QContinent.continent, sessionFactory);
    }

    public Optional<Continent> findById(final String id) {
        Continent continent = createQueryFactory()
                .selectFrom(QContinent.continent)
                .where(QContinent.continent.id.eq(id))
                .fetchOne();
        return Optional.ofNullable(continent);
    }
}
```

### Expected Behaviour

The `@Transactional` on class level should be propagated to all methods.

### Actual Behaviour

Compilation error:  `Error occurred writing META-INF files: Failed to load existing service definition files: javax.annotation.processing.FilerException: Attempt to reopen a file for path /build/classes/java/main/META-INF/services/io.micronaut.inject.BeanDefinitionReference`

Note when moving the `@Transactional` to the method the error is gone.

### Environment Information

- **Operating System**: OSX
- **Micronaut Version:** 1.0.0 M2
- **JDK Version:** JDK8
