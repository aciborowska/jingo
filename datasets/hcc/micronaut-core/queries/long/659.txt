The [TokenPropagationHttpClientFilter](
https://github.com/micronaut-projects/micronaut-core/blob/master/security/src/main/java/io/micronaut/security/token/propagation/TokenPropagationHttpClientFilter.java#L75) uses `ServerRequestContext.currentRequest()` to retrieve the original request and extract the token.


`ServerRequestContext.currentRequest()` is `Optional.empty()` if a client uses `Flowable` as a return type. Example:

```
@Client("http://localhost:8082")
@Requires(notEnv = Environment.TEST)
public interface BooksClient extends BooksFetcher {

    @Override
    @Get("/api/books")
    Flowable<Book> fetchBooks();
}
```

if it is `List<Book>`,  `serverRequestContext.currentRequest()` returns a request.

```
@Client("http://localhost:8082")
@Requires(notEnv = Environment.TEST)
public interface BooksClient extends BooksFetcher {

    @Override
    @Get("/api/books")
    List<Book> fetchBooks();
}
```

With this issue token propagation and multitenancy propagation do not work for `@Client` methods returning `Flowable`.

### Environment Information

- **Operating System**: Mac OS HighSierra
- **Micronaut Version:** 1.0.0.RC1
- **JDK Version:** 1.8.0_181
