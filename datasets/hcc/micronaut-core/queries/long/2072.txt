### Steps to Reproduce

1. Create a simple Micronaut app with spring-cloud-client included
2. Create a `bootstrap.yml` in `resources` and set `spring.cloud.config.uri` property there
3. Set the environment variable `SPRING_CLOUD_CONFIG_URI` before running the app
4. Add a breakpoint in `SpringCloudConfigurationClient#getPropertySources` and see the value of the `uri`.
5. If you use property `spring-cloud-config-uri` it gets replaced.

### Expected Behaviour

I would assume that environment variables take precedence over properties.

### Actual Behaviour

URI is not replaced with the environment variable value.

### Environment Information

- **Operating System**: Alpine linux and MacOS
- **Micronaut Version:** 1.2.0
- **JDK Version:** 1.8.0_144
