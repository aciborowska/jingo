

### Steps to Reproduce

1. Download sample application. Add Maven nature. Sample app contains the following annotation:
```
import java.lang.annotation.ElementType;
import java.lang.annotation.Target;

@Target({ElementType.PACKAGE, ElementType.TYPE, ElementType.FIELD, ElementType.CONSTRUCTOR, ElementType.METHOD})
@TestAnnotation
public @interface TestAnnotation { }
```
2. Set Annotation Processing Mode to "Automatically configure JDT APT"
3. Open the "Error Log" view.
4. Perfom a Maven > Update Project... (Alt+F5)

### Expected Behaviour

Should compile without errors.

### Further Explanation
This does not seem to be an issue when running maven directly from the command line.

I'm not sure why you would want an annotation that annotates itself, but it is valid Java. I found this issue when creating a factory with a bean of type [software.amazon.awssdk.http.SdkHttpClient](https://github.com/aws/aws-sdk-java-v2/blob/master/http-client-spi/src/main/java/software/amazon/awssdk/http/SdkHttpClient.java) and as you go up the hierarchy there is eventually the annotation [software.amazon.awssdk.annotations.SdkProtectedApi](https://github.com/aws/aws-sdk-java-v2/blob/master/core/annotations/src/main/java/software/amazon/awssdk/annotations/SdkProtectedApi.java) which annotates itself and seems to cause this issue.
