TLDR: https://github.com/micronaut-projects/micronaut-core/issues/2189 is not *really* fixed yet (in release 1.2.4). 
The `IllegalArgumentException` is now passed through pipeline and can be handled, but `ReadTimeoutException` still eventually occurs and is eventually thrown or logged.

To reproduce this bug, application (worker threads) must be running long enough that the timeout actually occurs.

This is *not* a cosmetic problem (ie low warning); complex pipelines still *fail* with `ReadTimeoutException`, but I am struggling to reproduce that in the demo app.

### Steps to Reproduce

https://github.com/podlesh/micronaut-http-client-bug-status

It is now updated to wait some some time at the end; this allows reactivex/netty workers to time out and report unhanded error.

### Expected Behaviour

No unhandled errors, no timeouts after `blockingGet`.
