### Steps to Reproduce
1. Java version -> complete
    1. Create Java app with swagger feature:
 `mn create-app swagger-java --lang=java --features swagger-java`
    2. Annotate the `Application` class with `@OpenAPIDefinition`
    3. Create an controller: 
`mn create-controller test`
    4. Assemble app:
 `./gradlew assemble`
    5. Check created swagger yaml file in `build/classes/java/main/META-INF/swagger/test-api-1.0.0.yml`: it contains both api doc and controller definition
2. Groovy version -> incomplete
    1. Create Groovy app with swagger feature: 
`mn create-app swagger-groovy --lang=groovy --features swagger-groovy` 
    2. Annotate the `Application` class with `@OpenAPIDefinition`
    3. Create an controller: 
`mn create-controller test`
    4. Assemble app:
 `./gradlew assemble`
    5. Check created swagger yaml file in `build/classes/groovy/main/META-INF/swagger/test-api-1.0.0.yml`: it contains only the api doc part

### Expected Behaviour

Groovy version of the swagger yaml file is equal to the java version

**Groovy:**

    openapi: 3.0.1
    info:
      title: Test API
      description: Test API blah
      version: 1.0.0

**Java:**
	
    openapi: 3.0.1
      info:
        title: Test API
        description: Test API blah
        version: 1.0.0
      paths:
        /test:
          get:
            operationId: index
            parameters: []
            responses:
        default:
          content:
            application/json:
              schema:
                $ref: '#/components/schemas/TestResponse'
      components:
        schemas:
          TestResponse:
            type: object
            properties:
        test:
          type: string
          nullable: false


### Actual Behaviour

Groovy version of the swagger yaml file is missing the controller part
