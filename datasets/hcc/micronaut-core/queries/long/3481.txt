With the following:

```java
@TypeHint(
    value = {UUID[].class, UUID.class},
    accessType = AccessType.ALL_PUBLIC_CONSTRUCTORS
)
```
If we try to get the `value`, for example in the `GraalTypeElementVisitor`: https://github.com/micronaut-projects/micronaut-core/blob/1.3.x/graal/src/main/java/io/micronaut/graal/reflect/GraalTypeElementVisitor.java#L103

The `UUID[]` is not returned, only `UUID`.

### Expected Behaviour

When retrieving the element information doing `final String[] introspectedClasses = element.stringValues(TypeHint.class);` it should return both `UUID[]` and `UUID`

### Actual Behaviour

It only returns `UUID`.

### Workaround

Declare the classes as String using `typeNames`

### Environment Information

- **Operating System**: Linux Mint
- **Micronaut Version:** 1.3.6 and 2.0.0
- **JDK Version:** 1.8
