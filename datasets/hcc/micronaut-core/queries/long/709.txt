### Steps to Reproduce

1. Create a _Groovy_ application with _Swagger_ feature :
```
mn create-app example-micronaut --features spock,groovy,swagger-groovy
```
2. Add `OpenAPIDefinition` annotation to the `Application` class.
https://docs.micronaut.io/latest/guide/index.html#openapi
```groovy
@OpenAPIDefinition(
        info = @Info(
                title = "Hello World",
                version = "0.0",
                description = "My API",
                license = @License(name = "Apache 2.0", url = "http://foo.bar"),
                contact = @Contact(url = "http://gigantic-server.com", name = "Fred", email = "Fred@gigagantic-server.com")
        )
)
@CompileStatic
class Application {
    static void main(String[] args) {
        Micronaut.run(Application)
    }
}
```
3. Compile with Gradle build task :
```
> Task :compileJava NO-SOURCE

> Task :compileGroovy
Note: Generating OpenAPI Documentation

> Task :processResources
> Task :classes
> Task :jar
[...]
``` 
4. OpenAPI logs that Documentation is generating but there is nothing in `build` directory

### Additional informations
On Java application generation is working with the following output that has the additional line of log `Note: Writing OpenAPI YAML to destination [...]`
```
> Task :compileJava
Note: Generating OpenAPI Documentation
Note: Writing OpenAPI YAML to destination: file:/[...]/example-micronaut-java/build/classes/java/main/META-INF/swagger/hello-world-0.0.yml

> Task :compileGroovy NO-SOURCE
> Task :processResources
> Task :classes
[...]
```

### Expected Behaviour

OpenAPI Documentation should be generated in the build folder.

### Actual Behaviour

OpenAPI Documentation is not generated in the build folder.
