Since Micronaut 1.3.2 (failing with 1.3.3 as well), when

1. **Given** a `HttpClientFilter` that injects 2 or more `Optional` properties as `@Value` (see `TwoOptionalsFilter` in the reproducer)
2. **When** having sources with enabled kotlin with Micronaut inject (this is why the reproducer is in `test-suite-kotlin`)
   ```
   kaptTest 'io.micronaut:micronaut-inject-java'
   ``` 
3. **And** when creating a `HttpClient` bean in such sources (see `CircularDependencyFailureWithOptionalsAndKaptTest` in the reproducer)
   ```
   applicationContext.createBean(HttpClient.class, new URL("http://localhost:80"))
   ```
4. **Then** `io.micronaut.context.exceptions.CircularDependencyException` is thrown when creating the `HttpClient` bean and the error is as follows:
