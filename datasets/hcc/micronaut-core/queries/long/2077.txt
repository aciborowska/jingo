This seems to be related to the `@MockBean` running for an Interface / Class that is using a generic method.  So I am adding the following:

```groovy
@MockBean(TestServiceImpl)
    TestService testService() {
        Mock(TestService)
    }
```

Which relates to the following interface, which has a Generic method. 

```java
public interface TestService {
    <T extends ListArguments> Mono<List<String>> findAll(T args);    
}
```

It seems this is related to the AST handling of this type of Java Generic method as if you change it to something without generics it compiles fine.  e.g.

```java
public interface TestService {
    Mono<List<String>> findAll(ListArguments args);
}
```
