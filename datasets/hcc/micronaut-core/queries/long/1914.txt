In `1.2.0.RC2`. For this controller:

```java
@Controller("/")
public class AppController {

	@Get("/rte")
	public Single<String> getRte() {
		return Single.create(emitter -> {
			emitter.onError(new RuntimeException());
		});
	}

	@Get("/fnfe")
	public Single<String> getFnfe() {
		return Single.create(emitter -> {
			emitter.onError(new FileNotFoundException());
		});
	}

	@Get("/ioe")
	public Single<String> getIoe() {
		return Single.create(emitter -> {
			emitter.onError(new IOException());
		});
	}

	@Get("/e")
	public Single<String> getE() {
		return Single.create(emitter -> {
			emitter.onError(new Exception());
		});
	}

}
```

The results of fetching data from particular endpoints is the following:

```
$ curl -m 5 http://localhost:8080/rte
{"message":"Internal Server Error: null"}

$ curl -m 5 http://localhost:8080/fnfe
curl: (28) Operation timed out after 5000 milliseconds with 0 bytes received

$ curl -m 5 http://localhost:8080/ioe
curl: (28) Operation timed out after 5000 milliseconds with 0 bytes received

$ curl -m 5 http://localhost:8080/e
{"message":"Internal Server Error: null"}
```

It looks for `IOException` and subclasses the operation is hanging endlessly.
