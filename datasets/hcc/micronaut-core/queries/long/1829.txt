Given a URI template with multiple variable expansions if the first one uses reserved expansion {+var} the other expansions do not get encoded. 

Adding this to the tests in UriTemplateSpec "Test expand URI template #template with arguments #arguments for path" 

'{+base}{hello}' | [base: 'http://example.com/home/', hello: "Hello World!"] | 'http://example.com/home/Hello%20World%21' 

fails with the result: 'http://example.com/home/Hello%20World!'
