The `@Client` annotation is removing a part of the URI.

The two fellowing codes are not giving the same result.

Working version:
```kotlin
@Client(value = ["http://api.com"], path = "/api/v1")
interface ManticoreClientService {
    @Get("/users")
    fun getUsers(): JsonNode
}

// GET http://api.com/api/v1/users
// client.getUsers() => [{}, ...]
```

Misbehaving version:
```kotlin
@Client("http://api.com/api/v1")
interface ManticoreClientService {
    @Get("/users")
    fun getUsers(): JsonNode
}

// GET http://api.com/users
// client.getUsers() => null
```

In the documentation we can read
```text
Recall that the value of @Client can be:

An absolute URI. Example https://api.twitter.com/1.1
...
```

I'm using:
Kotlin 1.2.41
Micronaut 1.0.0-SNAPSHOT
MacOS 10.13.4
