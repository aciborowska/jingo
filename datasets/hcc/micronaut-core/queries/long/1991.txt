### Steps to Reproduce

1. Create a Kotlin data class with at least one `Int` property, annotate it with `@Introspected`.
2. Use this data class as a query parameter on an endpoint.
3. Execute a valid request to that endpoint.

### Expected Behaviour

The parsing and binding should happen flawlessly.

### Actual Behaviour

The binding throws an exception:
```
Failed to convert argument [param] for value [null] due to: Value [[1]] cannot be converted to type : int
```

### Environment Information

- **Operating System**: MacOS
- **Micronaut Version:** 1.2.0
- **JDK Version:** 8

### Example Application

https://github.com/adamkobor/micronaut-pojo-query-binding-bug

### Short example

#### Data classes

```kotlin
@Introspected
data class IntrospectedQueryPojoWithIntParam(
    val param: Int
)

@Introspected
data class IntrospectedQueryPojoWithStringParam(
    val param: String
)

data class QueryPojo(
    var param: Int? = null
)

```

#### Controller
```kotlin
@Controller("/")
class ExampleController {

    // Failure: GET /introspected-int?param=1
    @Get("/introspected-int{?introspectedQueryParams*}")
    fun failingQueryBinding(introspectedQueryParams: IntrospectedQueryPojoWithIntParam): HttpStatus {
        return HttpStatus.OK
    }

    // Success GET /introspected-string?param=irrelevant
    @Get("/introspected-string{?introspectedQueryParams*}")
    fun successfulStringQueryBinding(introspectedQueryParams: IntrospectedQueryPojoWithStringParam): HttpStatus {
        return HttpStatus.OK
    }

    // Success GET /?param=1
    @Get("/{?queryParams*}")
    fun successfulQueryBinding(queryParams: QueryPojo): HttpStatus {
        return HttpStatus.OK
    }
}
```

### Additional info

It works perfectly in previous versions (in 1.2.0.RC2 too).
