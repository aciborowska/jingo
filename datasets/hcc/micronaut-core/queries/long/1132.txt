Hi,

I've notice a strange behaviour when dealing with type converters and http request parameter binding.

I've crafted an example project (linked below) that demonstrate the issue I found.

### Steps to Reproduce

1. Run the application
2. Perform a `GET /bar/forever`

### Expected Behaviour

The application should execute `FooBarController::bar` populating the `bar` argument accordingly and return

```json
{
  "value": "forever"
}
```
### Actual Behaviour

The application return an error 

```json
{
  "_links": {
    "self": {
      "href": "/bar/forever",
      "templated": false
    }
  },
  "message": "Required argument [Bar bar] not specified",
  "path": "/bar"
}
```

### Environment Information

- **Operating System**: Mac OS X 10.14 (Mojave)
- **Micronaut Version:** 1.0.3
- **JDK Version:** 1.8
