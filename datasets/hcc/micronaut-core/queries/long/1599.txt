The configuration for PropertyNamingStrategy doesn't appear to be picked up as advertised. I'm not seeing the CharSequencePropertyNamigStrategyConverter being utilized in an actual application. I've tried all possible ways to configure a YAML string that I am aware of. Each options produces a different Converter -- none of which seem to be the one which is intended.

## Using Single-quoted String

### Steps to Reproduce
1. include the below configuration in application.yml

```
jackson:
  property-naming-strategy: 'SNAKE_CASE'
```
### Expected Behaviour

SNAKE_CASE should be picked up by `CharSequencePropertyNamingStrategyConverter`.

### Actual Behaviour

ArrayNodeToIterableConverter is triggered instead.

