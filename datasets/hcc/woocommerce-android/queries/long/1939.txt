**Describe the bug**

When opening a "New Order" push notification, the app will open and load the Order Details. The header briefly shows up as "Order #null". When the app is opened, there's also a short glimpse of the order details that immediately disappears.

~I couldn't take a screenshot because it happens too fast.~ Please see this [video (22s)](https://drive.google.com/file/d/136qDrm_sr_tzqJpjPms43_7RAcCWlP9S/view?usp=sharing). 

<img src="https://user-images.githubusercontent.com/198826/74452897-e8450e00-4e3e-11ea-8796-044e0ffa330a.png" width="320">

**To Reproduce**
Steps to reproduce the behavior:

1. Use a slow internet connection. 
1. Create an order from the website. 
2. Wait for the push notification and tap it. 

**Expected behavior**

I expected a placeholder without the technical "null" message.

**Isolating the problem (mark completed items with an [x]):**

- [ ] I have deactivated other plugins and confirmed this bug occurs when only WooCommerce plugin is active.
- [ ] This bug happens with a default WordPress theme active, or [Storefront](https://woocommerce.com/storefront/).
- [x] I can reproduce this bug consistently using the steps above.

**Mobile Environment**

- Device: Google Pixel 3
- Android version: 10
- WooCommerce Android version: 3.6-rc-1
