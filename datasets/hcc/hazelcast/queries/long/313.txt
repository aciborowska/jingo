The group password has a default value, but it isn't optional. So the following is illegal:

 <group>
        <name>group2</name>
    </group>

According to the xsd it should be

 <group>
        <name>group2</name>
        <password>somepassword</password>
    </group>

I guess the password element in the xsd should be made optional.

[edit]
If you 'edit' this message, you can see the hidden xml. 

This can be done by this adding changing the xsd:

 <xs:element name="group">
        xs:complexType
            xs:sequence
                <xs:element ref="name"/>
                <xs:element ref="password" minOccurs="0" maxOccurs="1"/>
            /xs:sequence
        /xs:complexType
    /xs:element
