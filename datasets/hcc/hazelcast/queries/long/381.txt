I have the following setup - I am using TCP/IP configuration.
Notice that I am adding two members, one with valid hostname (validHost) and one with invalid hostname (invalidHost).
In the previous version of hazelcast -  we were using (1.9.4.6) it was throwing java.net.UnknownHostException but the cluster started correclty.
In 2.4.1. the cluster does not start.

<b>Configuration:</b>

``` java
Config cfg = new Config();
cfg.setProperty("hazelcast.jmx.detailed", "true");
cfg.setProperty("hazelcast.jmx", "true");
cfg.setProperty("hazelcast.prefer.ipv4.stack", "true");
cfg.setProperty("hazelcast.executor.client.thread.count", "20");
cfg.setProperty("hazelcast.version.check.enabled", "false");

NetworkConfig network = cfg.getNetworkConfig();
Join join = network.getJoin();
join.getMulticastConfig().setEnabled(false);

TcpIpConfig tcpConfig = join.getTcpIpConfig();
tcpConfig.addMember("validHost:7004");
tcpConfig.addMember("invalidHost:7004");
tcpConfig.setEnabled(true);

network.setPortAutoIncrement(false);
network.setPort(7004);

//cfg.setProperty("hazelcast.local.localAddress", "validHost"); // If I uncomment this one it works
cfg.getGroupConfig().setName("dev").setPassword("dev-pas");

HazelcastInstance instance = Hazelcast.newHazelcastInstance(cfg);
```
