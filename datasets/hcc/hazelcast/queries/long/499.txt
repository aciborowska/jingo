Problem with multiple HazelcastInstances on same machine and Client.

I have a HazelcastInstance with group Foo running on port 5701.
I have a HazelcastInstance with group Bar running on port 5702.
I have a Hazelcast client configured with group Bar and I use e.g. 127.0.0.1 without a port, the client will only try out 5701 and throw an authentication failure. It would be better if no authentication failure was thrown but 5702 would be tried. If that is not found... even a higher port number would be tried. Question of course is when to stop..  It would be nice a a client would be just as smart to discover a cluster, as the cluster members are among themselves.

I'm not sure if this would be considered a defect or just functionality missing.
