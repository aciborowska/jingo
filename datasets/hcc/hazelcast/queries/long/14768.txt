**Steps to reproduce:**
1) Prepare member application - you need just `HazelcastInstance hz = Hazelcast.newHazelcastInstance();` and configuration file like:
```
<?xml version="1.0" encoding="UTF-8"?>
<hazelcast
        xsi:schemaLocation="http://www.hazelcast.com/schema/config http://www.hazelcast.com/schema/config/hazelcast-config-3.10.xsd"
        xmlns="http://www.hazelcast.com/schema/config"
        xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">

    <properties>
        <property name="hazelcast.merge.next.run.delay.seconds">30</property>
        <property name="hazelcast.merge.first.run.delay.seconds">50</property>
        <property name="hazelcast.operation.backup.timeout.millis">1000</property>
    </properties>
    <advanced-network enabled="true">
        <member-server-socket-endpoint-config>
            <interfaces enabled="true">
                <interface>SETUP_INTERFACE_HERE</interface>
            </interfaces>
            <port auto-increment="true">5701</port>
        </member-server-socket-endpoint-config>
        <client-server-socket-endpoint-config>
            <interfaces enabled="true">
                <interface>SETUP_INTERFACE_HERE</interface>
            </interfaces>
            <port auto-increment="true">9000</port>
        </client-server-socket-endpoint-config>
    </advanced-network>
</hazelcast>
```

2) Prepare client application - you need just `HazelcastInstance client = HazelcastClient.newHazelcastClient();` and configuration file like:
```
<?xml version="1.0" encoding="UTF-8"?>
<hazelcast-client
        xsi:schemaLocation="http://www.hazelcast.com/schema/client-config http://www.hazelcast.com/schema/client-config/hazelcast-client-config-3.10.xsd"
        xmlns="http://www.hazelcast.com/schema/client-config"
        xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">
    <network>
        <cluster-members>
            <address>SETUP_MEMBER_ADDRESS_HERE</address>
            <address>SETUP_MEMBER_ADDRESS_HERE</address>
            <address>SETUP_MEMBER_ADDRESS_HERE</address>
        </cluster-members>
    </network>
</hazelcast-client>
```

3) Start cluster with 2 members on computerA and 1 member on computerB. Start client on computerA.
4) Disconnect computerB from network and wait than split brain is detected.
5) Connect computerB back to network and wait than members form a cluster.
6) See that cluster state is not updated on client. Restart client - you will see aforementioned NPE and client is not connected to cluster.
