3.6-122315-Enterprise:

On WAN Replication Q overrun, I expect to receive a WANReplicationQueueFullException, instead the client receives an UndefinedErrorCodeException.

Client code has to be:

``` java
            catch( UndefinedErrorCodeException e )
            {
                String originClassName = e.getOriginClassName();
                if( originClassName.equals( WANReplicationQueueFullException.class.getName() ) )
                {
                    log.warn( "WAN Replication Queue is full - throttling..." );
                    Thread.sleep( 5000 );
                }
            }
```

It would be more elegant if the WRQFException could be caught directly:

``` java
            catch( WANReplicationQueueFullException e )
            {
                log.warn( "WAN Replication Queue is full - throttling..." );
                Thread.sleep( 5000 );
            }
```
