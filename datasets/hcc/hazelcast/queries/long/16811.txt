**Describe the bug**
Into the file hazelcast-all-4.0.jar the class `com.hazelcast.web.WebFilterConfig` import the non existing class `com.hazelcast.util.StringUtil` 

**Expected behavior**
I want to use the `hazelcast-all-4.0.jar` to do a Filter Based Web Session Replication.

**To Reproduce**

Steps to reproduce the behavior:
1. Into web application put the `hazelcast-all-4.0` jars in your `WEB-INF/lib` folder.
2. Add the `com.hazelcast.web.WebFilter` file to the `web.xml` file as described in this link https://github.com/hazelcast/hazelcast-wm
