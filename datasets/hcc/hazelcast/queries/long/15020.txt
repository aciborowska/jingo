The following valid XML is rejected (valid according to XSD):

```xml
    <user-code-deployment enabled="true">
        <class-cache-mode/>
        <provider-mode/>
    </user-code-deployment>
```

The XSD specifies a default for them, but the parser throws:

It would also be fine if this configuration was allowed by XSD and used the defaults:
```xml
    <user-code-deployment enabled="true" />
```

Also the `hazelcast-full-example.xml` doesn't use the default value for `provider-mode`, which is weird. Also the default `hazelcast.xml` could contain that element with `enabled="false"`.
