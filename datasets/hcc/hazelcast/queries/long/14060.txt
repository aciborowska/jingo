I have a 3-node configuration.  No clients; all members.  Each member defines its own topic (like response-1, response-2, ...), including a single message listener.  I use HazelcastInstance.getTopic() to look up the topic when I need to send a response to a node.  Occasionally a response won't make it.  Given a node X, I'm doing:

`config.addTopicConfig( "response-X" );`
`config.addListener( new ListenerConfig( new MyMessageListener() ) );`

Then when node Y needs to send it a response:

`HazelcastInstance.getTopic( "response-X" ).publish( new Response() );`

If I change the configuration to define **all topics on all nodes**:

`config.addTopicConfig( "response-1" );`
`config.addTopicConfig( "response-2" );`
`config.addTopicConfig( "response-3" );`

... followed by adding the listener **after initialization**:

`HazelcastInstance hzInstance = Hazelcast.newHazelcastInstance( config );`
`hzInstance.getTopic( "response-X" ).addMessageListener( new MyMessageListener() );`

... then I no longer seem to experience message loss.
