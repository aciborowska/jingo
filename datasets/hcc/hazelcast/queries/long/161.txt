Using Hazelcast 2.1 (also verified in 2.0 & 2.0.2). When calling IMap.get(key) from within a transaction where the key is not in the map, causes the key to be **added** to the map on commit(). The value of the entry is null.

The output I see from the code below is:
    Size of map=1. Expected 0
    Value of key 1=null

To reproduce:

```
public class TxnGetTest
{
    public static void main(String[] args) {
        IMap<Long, Integer> map = Hazelcast.getMap("Test");

        Transaction txn = Hazelcast.getTransaction();
        txn.begin();
        map.get(1L);
        txn.commit();

        System.out.println("Size of map=" + map.size() + ". Expected 0");
        System.out.println("Value of key 1=" + map.get(1L));
        System.exit(0);
    }
}
```
