**Description**
When [hazelcast-client.zip](https://github.com/hazelcast/hazelcast/files/4734427/hazelcast-client.zip) is provided in spring boot application properties file & packaged as boot jar then the hazelcast provider fails to load the file as it doesn't allow "jar" URI scheme. Following is the method in **AbstractHazelcastCachingProvider** class that throws exception when the URI starts with "jar"

```java
    protected URL getConfigURL(URI location, @Nonnull ClassLoader classLoader)
            throws URISyntaxException, IOException {
        String scheme = location.getScheme();
        if (scheme == null) {
            // interpret as place holder
            location = new URI(System.getProperty(location.getRawSchemeSpecificPart()));
            scheme = location.getScheme();
        }
        URL configURL;
        if ("classpath".equals(scheme)) {
            configURL = classLoader.getResource(location.getRawSchemeSpecificPart());
        } else if ("file".equals(scheme) || "http".equals(scheme) || "https".equals(scheme)) {
            configURL = location.toURL();
        } else {
            throw new URISyntaxException(location.toString(), "Unsupported protocol in configuration location URL");
        }
        return configURL;
    }
```

**Expected behavior**

It should be able to handle the "jar" URI scheme.

**To Reproduce**

Steps to reproduce the behavior, assuming a spring boot application is available with required depedencies & [hazelcast-client.zip](https://github.com/hazelcast/hazelcast/files/4734427/hazelcast-client.zip):

1. Add spring.cache.type=jcache
2. Add spring.cache.jcache.provider=com.hazelcast.client.cache.impl.HazelcastClientCachingProvider
3. Add spring.cache.jcache.config=classpath:hazelcast-client.xml
4. Build a boot jar
5. Run the above jar

**Additional context**

- Java 8
- Spring Boot 2.2.6-RELEASE
- Hazelcast version server/client: 4.0.1
- [hazelcast-client.zip](https://github.com/hazelcast/hazelcast/files/4734427/hazelcast-client.zip)
