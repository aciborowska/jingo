When using the following implementation of `EntryProcessor`:
```
    public class EntryProcessorImpl extends AbstractEntryProcessor<String, String> {

        private final String value;

        public EntryProcessorImpl(String value) {
            this.value = value;
        }

        @Override
        public Object process(Map.Entry<String, String> entry) {
            entry.setValue(value);
            return null;
        }
    }

```

and invoking with `IMap.executeOnKey` map stats `Hit` count gets incremented by 2, not by 1. Same behavior is when you pull stats via both `IMap.getLocalMapStats` and Clustered REST API.
