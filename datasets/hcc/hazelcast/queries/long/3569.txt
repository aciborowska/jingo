When you try to inject a global serializer, like so:

`<hz:global-serializer implementation="someSerializerBean"/>`

Spring complains:

`org.springframework.beans.NotWritablePropertyException: Invalid property 'implementation' of bean class [com.hazelcast.config.GlobalSerializerConfig]: Bean property 'implementation' is not writable or has an invalid setter method. Does the parameter type of the setter match the return type of the getter?`

I guess it's because of the mismatch between `Serializer getImplementation()` and `GlobalSerializerConfig setImplementation(final ByteArraySerializer implementation)` or `GlobalSerializerConfig setImplementation(final StreamSerializer implementation)`. Also not sure whether setters are allowed to have a return type other than void in Spring.

Using Hazelcast 3.3
