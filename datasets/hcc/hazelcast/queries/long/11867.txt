Members don't check the group password anymore when they form a cluster (during joins). Nevertheless  for clients the check still exists.

This behavior should be changed for the client side too (i.e. group password is not checked).

```xml
<group>
    <name>dev</name>
    <password>this value should not be important anymore</password>
</group>
```

This issue is related to issue #11667 and PR #10377.
