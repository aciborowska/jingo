When a local `DataSerializable` class misses the default constructor, this is thrown:

```
Caused by: java.lang.NoSuchMethodException: Local classes can't conform to DataSerializable since they can't provide an explicit no-arguments constructor.
	at com.hazelcast.internal.serialization.impl.DataSerializableSerializer.tryClarifyNoSuchMethodException(DataSerializableSerializer.java:221)
```

This is true for a local class in non-static method. In a static method adding a no-arg constructor is possible.
