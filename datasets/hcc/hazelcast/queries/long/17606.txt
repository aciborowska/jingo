**Describe the bug**
Hazelcast 4.0.3 added the ability to set the expiration times for entries when using entry processors, with the addition of `ExtendedMapEntry`.
The suggested configuration does not update the TTL if the `EntryProcessor` implements the `Offloadable` interface.

**Expected behavior**
The entry expiration time should be updated to the new TTL.

**To Reproduce**

```java
private static class TTLChangingEntryProcessor<K, V> implements EntryProcessor<K, V, V>, Offloadable {

    private V newValue;
    private Duration newTtl;

    TTLChangingEntryProcessor(V newValue, Duration newTtl) {
        this.newValue = newValue;
        this.newTtl = newTtl;
    }

    @Override
    public V process(Entry<K, V> entry) {
        return ((ExtendedMapEntry<K, V>) entry).setValue(newValue, newTtl.toMillis(), TimeUnit.MILLISECONDS);
    }

    @Override
    public String getExecutorName() {
        return OFFLOADABLE_EXECUTOR;
    }

}
```
