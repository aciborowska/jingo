There are two issues actually.
1. The 3.2 documentation in "Spring integration" section (https://github.com/hazelcast/hazelcast/blob/master/hazelcast-documentation/src/SpringIntegration.md and
   http://www.hazelcast.org/docs/latest/manual/html-single/hazelcast-documentation.html#configuration ) gives example how to use hz:client

```
<hz:client id="client"
    group-name="${cluster.group.name}" group-password="${cluster.group.password}">
    <hz:member>10.10.1.2:5701</hz:member>
    <hz:member>10.10.1.3:5701</hz:member>
</hz:client>
```

And that syntax seems to be invalid according to the XSD. So both IDEA highlights everything in red as well as exception is raised at runtime:

```
Caused by: org.xml.sax.SAXParseException; lineNumber: 12; columnNumber: 94; cvc-complex-type.3.2.2: Attribute 'group-name' is not allowed to appear in element 'hz:client'.
```
1. Alternatively, the 3.1 documentation http://www.hazelcast.org/docs/3.1/manual/html-single/#SpringConfiguration gives a different syntax:

```
    <hz:client id="client">
        <hz:group name="${cluster.group.name}" password="${cluster.group.password}"/>
        <hz:network>
            <hz:member>10.10.1.2:5701</hz:member>
            <hz:member>10.10.1.3:5701</hz:member>
        </hz:network>
    </hz:client>
```

Which makes IDEA happy but still does not work. Here is the entire file

