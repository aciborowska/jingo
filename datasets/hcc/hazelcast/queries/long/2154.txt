We have seen an issue with incremental upgrade when changing from one version of Hazelcast to another - more specifically from 3.1.6 to 3.2.

In our case we have 6 Hazelcast clients running 3.1.6 and one client running 3.2 all trying to connect to a Hazelcast node that we have just upgraded to 3.2.

The issue is this: If a client running 3.1.6 connects to the node before a client running 3.2 connects, the node throws a HazelcastSerializationException.

If a client running 3.2 is the first to connect to the node, the connection is successful and clients running 3.1.6 are just rejected.

I have an example reproducing the issue at https://github.com/lbakman/spikes/tree/master/hazelcast-upgrade-issue

The examples can be run from the assembly/target/module\* folders.
