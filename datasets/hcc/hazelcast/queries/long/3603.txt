we have faced this issue with last two stable releases 3.3, 3.2.6. This is somehow connected to TransactionalList. Btw TransactionalList API is really poor compare to TransactionalMap.
This basic test case can simulate this error:

```
HazelcastInstance instance1 = Hazelcast.newHazelcastInstance();
TransactionContext tr = instance1.newTransactionContext();
tr.beginTransaction();
tr.getList("list").add("123");
tr.getList("list").add("1234");
tr.commitTransaction();
HazelcastInstance instance2 = Hazelcast.newHazelcastInstance();
```
