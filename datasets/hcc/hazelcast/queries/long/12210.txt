Cause: `PagingPredicate` iterationType` is null. The client set this param if the original predicate is a `PaginngPredicate`. If not, its null by default.

If the second instance changed to a member, it works since member checks first if its a `PartitionPredicate` & then creates a `Query` operation with correct iterationType.

Both `PartitionPredicate` & `PagingPredicate` can contain inner predicate. But for `PagingPredicate` it's optional. So

1 - On the client side, for all Predicate operations, the first check should be `PartitionPredicate` check. If it is, then inner predicate need to be compared to see if it's `PagingPredicate` or not.

2 - To prevent too many inner predicates, we could put controls on `PagingPredicate` to prevent inner predicate to be another `PagingPredicate` or `PartitionPredicate`.
