- a Hazelcast session ID is present in the request
- no local or clustered session exists associated with that ID

- the application's handling results in the response being committed without creating a session

- there is no Hazelcast or local session
- the response is sent without error

- WebFilter attempts to create a session after the response has been committed, resulting in an error.
