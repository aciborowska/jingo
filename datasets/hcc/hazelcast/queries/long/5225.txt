I encountered this bug on 3.5-SNAPSHOT. I tried 3.4 and 3.4.2.
Here's an test:

```
public void testPutTransientMapStoreWithMultipleInstance(){
        String mapName = randomMapName();
        TestHazelcastInstanceFactory nodeFactory = createHazelcastInstanceFactory(3);
        SimpleMapStore<Integer, Integer> mapStore = new SimpleMapStore<Integer, Integer>();

        Config cfg = new Config();
        MapStoreConfig mapStoreConfig = new MapStoreConfig();
        mapStoreConfig.setEnabled(true);
        mapStoreConfig.setImplementation(mapStore);
        mapStoreConfig.setWriteDelaySeconds(2);
        cfg.getMapConfig(mapName).setMapStoreConfig(mapStoreConfig);

        HazelcastInstance instance1 = nodeFactory.newHazelcastInstance(cfg);
        HazelcastInstance instance2 = nodeFactory.newHazelcastInstance(cfg);
        HazelcastInstance instance3 = nodeFactory.newHazelcastInstance(cfg);

        IMap map = instance1.getMap(mapName);
        map.putTransient(1, 1, 30, TimeUnit.SECONDS);

        sleepSeconds(4);

        assertNotEquals(map.entrySet(), mapStore.store.entrySet());
    }

```
