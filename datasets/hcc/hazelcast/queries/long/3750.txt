After upgrading to version 3.3.1 an `UnsupportedOperation` is thrown when loading the cache with object that has an index on a `Portable` field.

The index is configured as follows:

```
<map name="...">
  ...
  <indexes>
    <index ordered="false">identity.foreignKey</index>
  </indexes>
</map>
```

With the code looking like this:

```
public class ParentIdentity implements Portable
{
 ...
}

public class Parent implements Portable
{
  private ParentIdentity identity; 
  ...
}

public class ChildIdentity implements Portable
{
  private ParentIdentity foreignKey;
  ...
}

public class Child implements Portable
{
  private ChildIdentity identity; 
  ...
}
```

Tracing through the code it looks as though the correct `FieldDefinition` is being found in the `PortableExtractor` however the `PortableFieldExtractor` being returned is a `PortableUnsupportedFieldExtractor`, which throws the `UnsupportedOperation` when trying to get the attribute type.

I am running the cache in it's on JVM. the load being performed from another JVM and the objects are all in the same factory.
