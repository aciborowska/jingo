I have 2 members configured in a cluster. 

Member 1 is in a loop
- Creates jcache (with unique name)
- sleep 1 sec
- Schedules destroy of cache 10 secs later
- sleep 1 sec

Member 2
- starts up
- sleeps for 10 secs
- shuts down

Occasionally member 2 will raise the "CacheNotExistsException" for one or more of the active jcaches created by Member 1 during cluster synchronization(?)

I've been running on Linux, and issue occurs on both: centos 6 and Ubuntu 16.4.

For reproduction, I've attached a test program (jcache-versions.tar.gz
[jcache-versions.tar.gz](https://github.com/hazelcast/hazelcast/files/1217312/jcache-versions.tar.gz)
) that should reproduce the issue

1. Unpack jcache-versions.tar.gz test program on two linux machines with Java 8.
2. Edit conf/cache_versions_hazelcast.xml on both machines, to configure host names/ip addresses for the cluster
3. On machine 1 - bin/jcache-versions.bash
