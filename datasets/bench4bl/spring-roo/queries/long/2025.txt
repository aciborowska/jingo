The delete method in the controller generated by Roo has a problem: it generates a redirect with duplicate size and page parameterss. E.g. /pets?page=1&size=10&page=1&size=10
The generated code is:
        model.addAttribute("page", (page == null) ? "1" : page.toString());
        model.addAttribute("size", (size == null) ? "10" : size.toString());
        return "redirect:/pets?page=" + ((page == null) ? "1" : page.toString()) + "&size=" + ((size == null) ? "10" : size.toString());
A solution is to return "redirect:/pets", like list method.
