DefaultXmlBeanDefinitionParser calls preProcessXml() before setting the values for defaultLazyInit, defaultAutowire and defaultDependencyCheck.  Currently, classes extending from DefaultXmlBeanDefinitionParser can override the methods for getDefaultAutowire and getDefaultDependencyCheck, and calls to the parseBeanDefinitionElement() method will use the overridden getters.  However, parseBeanDefinitionElement(), does not use the getDefaultLazyInit() method to get the defaultLazyInit value.  Therefore, classes extending from DefaultXmlBeanDefinitionParser cannot set a value for defaultLazyInit before calling parseBeanDefinitionElement().  
Possible solutions: 
  Add setters for these three values.
  Set the values before calling preProcessXml()
  Make parseBeanDefinitionElement() use the getDefaultLazyInit() method instead of the raw value
