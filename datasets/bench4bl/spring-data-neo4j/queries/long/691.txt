Spring Data Neo4j 3.3.1 supports Neo4j 2.2, but it appears to have some runtime dependency on a class that is not present in Neo4j 2.2.3.  The class that is missing is QueryNotPossibleException, which was in neo4j-lucene-index 2.2.0 but seems to have been removed by 2.2.3.  
When our application starts, I get the following error if we depend on SDN 3.3.1 and Neo4j 2.2.3, but not if we depend on SDN 3.3.1 and Neo4j 2.2.0.  (Sorry for redundancy; just trying to be clear.)
