Query parameters such as '?page.sort=name&page.sort=description' or '?page.sort=name,description' end up creating a Sort instance (via org.springframework.data.web.PageableArgumentResolver.SortPropertyEditor.setAsText(String)) that sorts on SINGLE property named 'name,description' instead of TWO properties named 'name' and 'description'. This results in 'org.springframework.data.mapping.PropertyReferenceException: No property name,description found for type com.co.Xyz'. Fix might to to update line in setAsText method from 'setValue(new Sort(order, text));' to 'setValue(new Sort(order, StringUtils.commaDelimitedListToStringArray(text)));'.
