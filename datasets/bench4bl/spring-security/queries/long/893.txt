The methods in SavedRequestAwareWrapper treat parameters differently. If the method getParameter(String) returns the parameter value from the wrapped request OR the saved request, while the methods getParameterMap() and getParameterValues(String) returns values from the wrapped request AND the saved request.
I think the correct thing to do is to always return parameter values from the wrapped request OR the saved request, giving the wrapped request precedence.
This bug corrupt requests since single valued parameters become multi valued (duplicate value) if the wrapped request is the same as the saved request.
I found a (kinda) workaround in schema based configuration setting servlet-api-provision attribute to false......
