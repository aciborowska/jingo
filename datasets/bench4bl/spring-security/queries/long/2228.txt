In 3.2.0.M2, the spring-security-config bundle has a mandatory import statement for org.openid4java.consumer: org.openid4java.consumer;version="[0.9.5, 1.0.0)". This should have resolution:=optional. Without optional resolution, OSGi users are forced to include a large number of additional dependencies despite not using openid4java. Further, openid4java's bundle in EBR doesn't work with Servlet 3.
